# 内容森林 - 果实模块设计 (Fruit Module Design)

> 状态：MVP 阶段设计
> 日期：2026-03-19

## 1. 核心设计理念 (Core Concepts)

果实（Fruit）是内容森林中，种子经过 AI Agent 生成后的具体内容产物。果实模块的设计核心围绕**“溯源性”**、**“进化树展现”**与**“平台解耦”**展开。

*   **树状物理法则**：果实是进化树（DAG，有向无环图）上的节点，不设隐藏/归档状态，保留完整的繁衍记录，被淘汰的果实以视觉“枯萎”展现。
*   **读时模式 (Schema-on-read)**：系统不硬编码平台数据指标，由数据监控器提供自描述的数据反馈载荷。
*   **画布即工作台**：抛弃传统的列表与侧边栏导航，将进化树的无限画布作为唯一核心工作台。

## 2. 数据结构设计 (Data Structure)

果实采用**“容器化”设计 (Container Pattern)**，将系统通用字段与生成器业务字段分离。

### 2.1 核心数据结构

```typescript
interface Fruit {
  // --- 基础身份信息 ---
  id: string;
  userId: string;
  seedId: string; // 所属种子
  
  // --- 进化溯源 ---
  parentFruitIds?: string[]; // 父节点（支持多父节点，为后续嫁接重组预留）
  generation: number;        // 代数（种子产出为1，基于果实迭代为2+）
  
  // --- 生产上下文 ---
  // 记录生成时的"快照"，不记录冗长Prompt，只记录影响因子
  context: {
    generatorId: string;
    generatorVersion: string;
    nutrientsUsed?: string[]; // 当时使用的营养库版本/ID
  };

  // --- 极简状态机 ---
  // 废弃了归档(archived)和生成中(ripening)，保持进化树完整且契合前后端分离逻辑
  status: 'generated' | 'picked' | 'rejected' | 'published';
  createdAt: number;
  updatedAt: number;

  // --- 基因突变记录 ---
  mutation: {
    applied: boolean;
    type?: 'style' | 'remix' | 'anti-logic';
    description?: string; 
  };

  // --- 最小展示契约 ---
  preview: {
    title: string;
    summary: string;
    coverUrl?: string; // 封面图URL
    contentType: 'text' | 'image_text' | 'video_script' | 'audio';
    platform: string;  // 目标平台
  };

  // --- 自由载荷 ---
  // 媒体资源只存URL，不存Base64，由Agent在生成时上传OSS
  payload: Record<string, any>;
}
```

### 2.2 数据反馈的领域解耦 (Fruit Growth Data)

数据反馈（Metrics）本质上属于“监控器领域”，它是果实发布后产生的衍生数据。为了遵循单一职责原则，将数据反馈从果实实体中剥离，作为一个独立的实体附着于果实之上。

```typescript
// 独立的成长数据实体
interface FruitGrowthData {
  fruitId: string;       // 关联的果实 ID
  platform: string;      // 冗余平台信息，方便监控器独立工作
  collectedAt: number;   // 数据最后收集/填写的更新时间
  metrics: MetricsField[]; // 自描述的数据载荷
}

// 灵活的指标载荷
interface MetricsField { 
   key: string;         // 字段唯一键名 (例如: 'coin', 'retention_rate')
   value: number;       // 可量化的数据值 (例如: 150, 0.45)
   label: string;       // 页面展示名称 (例如: '投币数', '完播率')
   description: string; // 字段描述，供Agent营养汲取时理解数据意义
}
```

### 2.3 平台指标字典 (Platform Metrics Dictionary)

为了解决用户手动填写的痛点，系统引入“平台指标字典”。
1.  **字典预设**：系统预设各平台的常见指标（如小红书的浏览、点赞、收藏）。
2.  **按需勾选**：用户手动录入时，先从字典中勾选关心的指标，系统自动带出 `label` 和 `description`，用户只需填写 `value` 数字即可。
3.  **动态组装**：前端将用户填写的数字与字典的描述组装成完整的 `MetricsField[]` 提交给后端。这保证了底层数据的自描述性，对 Agent 汲取营养极其友好。

## 3. 核心功能与流转 (Core Functions)

1.  **生成 (Generate)**：由 AI IDE 中的 Agent 驱动，基于种子/父果实生成新果实，结果通过 MCP 落库。
2.  **选品比对 (Taste/Compare)**：提供对比视图，用户可将画布上的多个果实拖拽至选品槽，横向对比其 Payload 内容。
3.  **修剪 (Pruning)**：原位编辑（In-place Edit）。不新开迭代分支，保持 MVP 极简，用户直接修改果实文本。
4.  **进化 (Evolution)**：
    *   **营养汲取 (Extract)**：针对 Picked（优质）果实，提取成功经验，沉淀到对应的营养库。
    *   **堆肥 (Compost)**：针对 Rejected（淘汰）果实，提取失败教训，作为负向经验沉淀到营养库。

## 4. API 接口设计 (API Interfaces)

为了保持职责单一，API 严格分为 **MCP (供 AI Agent 调用)** 和 **HTTP (供 Web UI 调用)** 两套。

### 4.1 MCP Tools (Agent 操作层)
用于 Agent 读写存储层数据：
*   **`save_fruits`**: 核心写入，批量存入新生成的果实。入参包含 `seedId`, `parentFruitIds`, `context`, `fruits` 列表。
*   **`get_fruit_context`**: 获取果实完整上下文，供 Agent 基于此迭代或汲取。
*   **`update_fruit_status`**: 更改果实状态（如 Agent 自动标记为 rejected）。
*   **`compost_fruit`**: 写入堆肥总结的负向经验。

### 4.2 HTTP RESTful API (Web 前端操作层)
处理用户在页面的直接操作：

#### 1. 查询种子下所有果实 (List Fruits by Seed)
- **URL**: `GET /api/seeds/{seedId}/fruits`
- **Headers**: `X-User-Id: {userId}`
- **Response**: 
  ```json
  {
    "code": 0,
    "data": {
      "list": [
        {
          "id": "fruit_abc123",
          "seedId": "seed_xyz789",
          "parentFruitIds": [],
          "status": "generated",
          "preview": { ... },
          "payload": { ... },
          "metrics": []
        }
      ]
    }
  }
  ```

#### 2. 手动选中果实 (Pick Up Fruit)
- **URL**: `PATCH /api/fruits/{fruitId}/pickup`
- **Headers**: `X-User-Id: {userId}`
- **Response**: 
  ```json
  { "code": 0, "data": { "id": "fruit_abc123", "status": "picked" } }
  ```

#### 3. 淘汰并堆肥果实 (Compost Fruit)
- **URL**: `PATCH /api/fruits/{fruitId}/compost`
- **Headers**: `X-User-Id: {userId}`
- **Response**: 
  ```json
  { "code": 0, "data": { "id": "fruit_abc123", "status": "rejected" } }
  ```

#### 4. 原位修剪果实 (Prune Fruit)
- **URL**: `PUT /api/fruits/{fruitId}/payload`
- **Headers**: `X-User-Id: {userId}`
- **Body**: 
  ```json
  { 
    "preview": { "title": "修改后的标题", ... }, 
    "payload": { "content": "修改后的正文" } 
  }
  ```
- **Response**: 
  ```json
  { "code": 0, "data": { "id": "fruit_abc123" } }
  ```

#### 5. 获取果实成长数据 (Get Fruit Growth Data)
- **URL**: `GET /api/fruits/{fruitId}/metrics`
- **Headers**: `X-User-Id: {userId}`
- **Response**: 
  ```json
  {
    "code": 0,
    "data": {
      "fruitId": "fruit_abc123",
      "platform": "xiaohongshu",
      "collectedAt": 1710830000000,
      "metrics": [
        { "key": "views", "value": 12000, "label": "浏览量", "description": "内容曝光次数" }
      ]
    }
  }
  ```

#### 6. 更新果实成长数据 (Update Fruit Metrics)
- **场景**: 供前端用户手动录入提交，或未来监控器自动写入。
- **URL**: `PUT /api/fruits/{fruitId}/metrics`
- **Headers**: `X-User-Id: {userId}`
- **Body**: 
  ```json
  {
    "metrics": [
      { "key": "views", "value": 12500, "label": "浏览量", "description": "内容曝光次数" },
      { "key": "likes", "value": 350, "label": "点赞量", "description": "用户喜欢次数" }
    ]
  }
  ```
- **Response**: `{ "code": 0, "data": { "success": true } }`

## 5. 前端页面设计：无限画布工作台 (The Infinite Canvas)

**核心理念：进化树不是左侧导航，进化树就是整个工作台。**
### 5.1 视觉与交互构想

**1. 入场动画 (Growth Animation)**
*   **触发**：从种子库点击某颗种子卡片进入。
*   **转场**：页面通过共享元素动画平滑过渡，背景变暗进入“实验室”模式。
*   **生长**：种子节点在画布中央落定发光。连线如藤蔓般向上/四周蔓延（SVG path `stroke-dasharray` 动画）。连线到达终点时，果实节点伴随 Spring 弹簧动效依次弹现。

**2. 核心工作台布局**

```text
=============================================================================
[ 顶部极简导航: 🌲 Content Forest  |  当前种子: AI工具推荐  |  视图缩放: 80% ]
=============================================================================

                   (全屏的无限画布、支持鼠标拖拽平移、滚轮缩放)
                   
                           🌱 [种子节点]
                                │
                 ┌──────────────┴──────────────┐
                 │                             │
        (枯萎的灰节点)                    (高亮的红节点)
        [🍎 果实 A (淘汰)]               [🍎 果实 B (Pick!)] 👈 (鼠标悬浮 Hover)
                 │                             │          +---------------------+
                 │                             │          | ✨ 进化 (汲取/堆肥) |
          [🍎 果实 A-1]                        │          | 📝 修剪 (编辑)      |
                                               │          | 🚀 发布             |
                                               │          +---------------------+
                                               │
                                       [🍎 果实 B-1]

=============================================================================
[ 底部/右侧 悬浮吸边工具栏 ]
   [+ 唤起生成器]  |  [⚖️ 选品比对槽 (0/3)] 
=============================================================================
```

**3. 右侧详情抽屉 (Detail Drawer)**
当点击某个果实节点（非悬浮操作按钮）时，节点平移至屏幕偏左，右侧滑出毛玻璃质感的详情抽屉。抽屉采用双 Tab 设计解耦内容与数据：


### 5.2 核心微交互 (Micro-interactions)
*   **卡片悬浮操作 (Popover)**：点击或 Hover 树节点，就地弹出操作轮盘（Pick、修剪、进化）。
*   **拖拽对比选品 (Drag & Drop)**：点击工具栏的 `[⚖️ 选品比对槽]` 滑出抽屉，用户可直接从树上拖拽果实进入槽内并排对比。
*   **修剪反馈**：原位展开文本框编辑，完成后节点闪烁确认。

### 5.3 技术选型
*   **核心画布**：`Vue Flow`（支持无极缩放、拖拽平移、自定义节点、连线动效）。
*   **动效引擎**：`GSAP` 或 `@vueuse/motion`（采用带有弹簧阻尼的物理动效，拒绝生硬的线性动画）。
*   **样式框架**：定制版 `TailwindCSS`（禁用默认色板，深度定制专属 UI）。

### 5.4 美学与配色设定 (反主流风格)
*   **背景**：深岩灰 (`#1A1A1A`) 或墨绿黑 (`#0F1712`)，配合细微噪点（Noise）纹理与极细网格线，凸显“实验室/工作台”的硬核感。
*   **节点材质**：毛玻璃（Glassmorphism）叠加微妙内阴影，使卡片在画布上产生浮起感。
*   **状态高亮**：
    *   **Picked (高光)**：使用高饱和度“荧光绿”或“日落橙”描边，带轻微呼吸发光动效。
    *   **Rejected (堆肥)**：色彩饱和度降至 20%，透明度 50%，呈现“枯萎/蒙尘”的视觉感。

### 5.5 核心区域交互拆解

**① 画布节点 (Fruit Node Card)**
*   **常态**：只展示基础 `preview` (平台 Icon、标题、摘要)。
*   **悬浮 (Hover)**：卡片微微放大（Spring 动效），四周浮现操作按钮：`✨ 汲取灵感`、`✂️ 动手修剪`、`🗑️ 丢进堆肥`。
*   **修剪模式 (Edit)**：点击修剪，卡片通过 3D 翻转动画变为沉浸式无边框输入框，回车确认保存。

**② 对比选品槽 (Tasting Dock)**
*   平时隐藏于底部，当拖拽果实节点时，底部滑出带有磁吸反馈的 Dock 区。
*   拖入 2-3 个果实后展开 Dock，变为横向宽屏滚动区，平铺展示各个果实的 `payload`。支持在对比视图中直接操作 Pick。

**③ 连线动效 (Edges)**
*   采用带流动粒子的贝塞尔曲线 (Bezier) 表现遗传关系。
*   “突变”生成的子果实连线呈现为断续虚线或电流感，视觉上暗示基因变异。
