基因汲取浮动面板

在事件发生时记录原因、给出依据、输出可沉淀的建议，而不是把整个能力塞进一个抽屉。

Seed 壁纸产品宣传种子 正在沉淀情绪价值、收藏感和低噪声表达。
Fruit · selected 先讲情绪收益，再讲产品本体 点击选择时，输入“为什么选择”，作为汲取事件的上下文。
Fruit · eliminated 直接罗列规格和参数 淘汰时输入“为什么淘汰”，系统提取反向规避基因。
Fruit · candidate 收藏感的视觉切片 这类节点可以继续观察，暂不触发汲取也能保留在待处理队列。
Fruit · verified 发布后数据表现稳定 发布验证与反馈回流可作为更高权重证据源。
设计意图

把基因汲取从“操作抽屉”改成“浮动任务面板”，让它更像工作区里的即时决策层，而不是一个第二页面。

请在触发汲取时填写“为什么选择 / 为什么淘汰 / 为什么发布验证成功”，这些原因会成为后续提取基因的重要上下文。

事件原因输入

事件上下文
事件类型 选择 / 淘汰 / 发布验证 / 数据反馈
证据强度 弱证据、强证据、跨样本复现
归因目标 情绪表达、结构、标题、CTA、节奏
汲取依据 Evidence-backed
1 有效性 不是“看起来像”，而是与结果有稳定关联
2 可迁移性 能否跨种子、跨平台、跨样本复现
3 可作用性 能否转成生成器可执行的约束或提示
候选基因：先给用户收益，再给产品本体 0.86 confidence
  • 来源：3 个被选择果实 + 1 次发布验证
  • 方法：事件归纳 + 互信息筛选 + 人工确认
  • 用途：作为开头结构基因注入生成器提示
