## Why

工作区前端已经具备树状画布、节点交互、悬浮输入框和资源引用等 UI/UX 设计，但后端目前缺少一个面向工作区首屏加载的统一聚合入口。若前端直接分别调用种子、果实、生长、生成器、营养库、基因库等领域接口拼装工作区，会造成接口调用分散、加载状态难以统一、领域边界在前端被迫外泄。

该变更用于新增轻量化的工作区聚合能力：它只读取并组合现有领域事实，不持久化画布节点位置，不替代任何领域模块的业务职责。

## What Changes

- 新增工作区后端聚合模块，用于根据种子读取工作区所需的树结构、节点摘要、边关系和工作区只读状态。
- 新增工作区快照 API 契约，对应一个独立的工作区 Controller 文档，服务工作区首屏加载和刷新。
- 聚合节点的生长状态与最近失败输入提示，供前端展示节点生长动效、禁止重复生长、恢复最近失败输入。
- 聚合工作区可引用资源，包括可用生成器、可引用营养内容和可引用基因洞察，供悬浮输入框和枝化详情面板使用。
- 工作区模块不新增数据库表，不保存节点拖拽位置、画布视口、布局坐标或任何工作区私有状态。
- 工作区模块不接管突变操作：果实选择/淘汰/恢复、枝化生长、发布验证、数据回流和基因汲取仍由各自领域接口负责。

## Capabilities

### New Capabilities

- `workspace-aggregation`: 定义工作区后端聚合读取能力，覆盖工作区快照、内容树节点与边关系、节点生长状态、失败输入提示、可引用资源聚合和只读工作区语义。

### Modified Capabilities

无。

## Impact

- 影响后端模块：新增 `workspace` 聚合模块及其应用服务、HTTP Controller 和模块导出。
- 影响 API 契约：需要在 `docs/api/` 新增工作区 Controller 对应的 OpenAPI 文档，遵守“一份 `.yaml` 对应一个 Controller 文件”的约束。
- 不影响 SQL 契约：该变更不新增表、不改字段、不维护工作区私有持久化状态，因此不新增 `docs/sql/workspace.sql`。
- 依赖现有领域能力：种子、果实、枝化生长、生成器、营养库、基因库、发布验证等领域继续作为事实来源。
