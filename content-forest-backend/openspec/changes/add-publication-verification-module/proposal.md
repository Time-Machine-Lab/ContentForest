## Why

内容森林第一期需要把“已选择果实”送入真实外部环境验证，但当前后端尚无发布验证模块，无法记录某个果实被发布到哪个外部目标，也无法为后续数据回流提供发布记录入口。

该变更依据 `docs/design/domain/发布验证领域模块设计文档.md`、`docs/design/domain/果实领域模块设计文档.md`、`docs/design/内容森林架构设计文档.md` 和 `docs/内容森林第一期开发规划文档.md`，先实现第一期人工发布验证闭环，为后续数据回流领域开发提供可追溯基础。

## What Changes

- 新增发布验证后端模块，围绕发布记录管理第一期人工发布验证事实。
- 新增人工发布器能力：第一期只记录用户已在外部平台完成发布的事实，不调用任何外部平台 API。
- 新增发布记录创建能力：只有已选择果实才能创建发布记录；创建时记录发布目标、发布凭证和发布备注；发布时间跟随发布记录创建时间。
- 新增发布记录编辑能力：允许编辑发布目标、发布凭证和发布备注；不允许修改关联果实；不提供修改发布时间能力。
- 新增发布记录查询能力：支持查看单条发布记录，并支持按果实列出该果实的多条发布记录。
- 新增数据回流前置能力：提供发布记录存在性校验或读取能力，供后续数据回流模块基于发布记录追加快照。
- 新增果实可发布校验端口：发布验证模块不直接管理果实状态，只依赖窄端口校验果实是否可发布；真实果实模块集成可在后续提案中完成。
- 新增 `docs/api/publication.yaml` 与 `docs/sql/publication.sql` 顶层契约文档，一份 API 契约对应一个 Publication Controller，一张发布记录表对应一个 SQL 文档。
- 不新增删除、归档、平台枚举、发布器管理页面、自动发布器、发布表现数据、截图附件上传或数据快照能力。

## Capabilities

### New Capabilities

- `publication-verification`: 管理人工发布器和发布记录，支持为可发布果实创建、编辑、查看和按果实列出发布记录，并为数据回流提供发布记录入口。

### Modified Capabilities

无。

## Impact

- 影响后端模块层：新增 `src/modules/publication/`，包含领域类型、应用服务和模块出口。
- 影响存储层：新增发布记录存储端口、内存适配器和 SQLite 适配器。
- 影响接口层：新增 Publication Controller，对应 `docs/api/publication.yaml`。
- 影响顶层契约：新增 `docs/api/publication.yaml` 和 `docs/sql/publication.sql`。
- 影响应用装配：需要在后端启动装配中注册发布验证服务、存储适配器和控制器。
- 影响测试：需要覆盖发布记录创建、编辑、查询、按果实列表、果实可发布校验失败、关联果实不可变、无删除归档能力等核心路径。
- 依赖果实模块：发布验证模块需要果实可发布校验端口；在真实果实模块完成前，可使用测试替身或临时适配器支撑模块测试，不在本变更中实现果实领域真实能力。
