## ADDED Requirements

### Requirement: 收敛中断遗留生长任务
系统 SHALL 在单实例后端运行模型下识别并收敛没有真实执行者的遗留生长任务。系统 MUST 不自动重试这些任务，MUST 根据已有尝试结果将任务收敛为已完成或失败，并 MUST 释放对应来源节点生长锁。

#### Scenario: 启动时收敛已有成功尝试的遗留任务
- **WHEN** 后端应用启动时发现数据库中存在生成中生长任务
- **AND** 该任务不在当前进程真实执行集合中
- **AND** 该任务已有至少一个成功的内部果实生成尝试
- **THEN** 系统 MUST 将该任务标记为已完成
- **AND** 系统 MUST 根据成功尝试汇总已创建果实引用
- **AND** 系统 MUST 释放该任务的来源节点生长锁

#### Scenario: 启动时收敛没有成功尝试的遗留任务
- **WHEN** 后端应用启动时发现数据库中存在生成中生长任务
- **AND** 该任务不在当前进程真实执行集合中
- **AND** 该任务没有任何成功的内部果实生成尝试
- **THEN** 系统 MUST 将该任务标记为失败
- **AND** 系统 MUST 保存该任务最近失败输入，供用户手动重试
- **AND** 系统 MUST 释放该任务的来源节点生长锁

#### Scenario: 不收敛当前真实执行中的任务
- **WHEN** 系统检查生成中生长任务
- **AND** 该任务仍在当前进程真实执行集合中
- **THEN** 系统 MUST 保持该任务生成中状态
- **AND** 系统 MUST 不释放该任务来源节点生长锁

## MODIFIED Requirements

### Requirement: 维护来源节点生长锁
系统 SHALL 在生长任务运行期间为来源节点维护生长锁。同一来源节点已有真实进行中任务或有效生长锁时，系统 MUST 拒绝再次发起生长；其他未锁定节点 MUST 不受影响。对于服务中断后遗留且没有真实执行者的生长锁，系统 MUST 通过中断遗留任务收敛释放该锁。

#### Scenario: 创建任务时锁定来源节点
- **WHEN** 系统成功创建生长任务
- **THEN** 系统 MUST 将该任务来源节点标记为正在生长

#### Scenario: 同一节点重复生长
- **WHEN** 用户对一个正在生长的来源节点再次发起枝化生长
- **THEN** 系统 MUST 拒绝创建新的生长任务
- **AND** 系统 MUST 返回节点正在生长的状态信息

#### Scenario: 其他节点可以继续生长
- **WHEN** 某个种子或果实节点正在生长
- **THEN** 系统 MUST 允许用户对其他未锁定且可生长的节点发起枝化生长

#### Scenario: 任务结束释放锁
- **WHEN** 生长任务进入已完成或失败状态
- **THEN** 系统 MUST 释放该任务来源节点的生长锁

#### Scenario: 异常路径释放锁
- **WHEN** 生长任务执行过程中发生未预期异常
- **THEN** 系统 MUST 将任务标记为失败或可感知的失败状态
- **AND** 系统 MUST 释放该任务来源节点的生长锁

#### Scenario: 释放中断遗留锁
- **WHEN** 系统发现来源节点生长锁关联的任务没有真实执行者
- **THEN** 系统 MUST 先收敛该生长任务状态
- **AND** 系统 MUST 释放该来源节点生长锁

### Requirement: 通过 AgentPort 执行果实生成尝试
系统 SHALL 通过 AgentPort 调用枝化生长 Agent 能力。果实数量为 N 时，系统 MUST 在一个生长任务内执行 N 次内部果实生成尝试；系统 MUST 使用受限并发的生产者/消费者模式执行这些尝试，默认同时运行的尝试数为 2，且最大并发数 MUST NOT 超过 3。

#### Scenario: 数量为五时执行五次尝试
- **WHEN** 用户发起果实数量为 5 的枝化生长任务
- **THEN** 系统 MUST 在该任务内执行 5 次内部果实生成尝试
- **AND** 系统 MUST 为每次尝试调用 AgentPort 的 growth 任务能力

#### Scenario: 受限并发执行多次尝试
- **WHEN** 生长任务包含多个内部果实生成尝试
- **THEN** 系统 MUST 通过 worker 消费待生成尝试队列
- **AND** 同一任务内同时运行的尝试数 MUST 不超过系统并发上限

#### Scenario: AgentPort 接收任务上下文和授权范围
- **WHEN** 系统调用 AgentPort 执行果实生成尝试
- **THEN** 系统 MUST 传递任务类型、来源节点引用、用户输入、生成器引用、授权资源范围和本次尝试目标
- **AND** 系统 MUST 不直接把来源 Markdown 全文或真实文件路径作为必需输入塞给 AgentPort

#### Scenario: 突变概率第一期不传递
- **WHEN** 用户提交枝化生长请求
- **THEN** 系统 MUST 不要求突变概率参数
- **AND** 系统 MUST 不把突变概率作为第一期 AgentPort 必需输入

### Requirement: 判定任务完成或失败
系统 SHALL 根据成功创建并挂载的果实数量判定生长任务结果。至少一个果实成功创建时，任务 MUST 标记为已完成；没有任何果实成功创建时，任务 MUST 标记为失败。部分成功 MUST 不回滚已创建果实。并发执行时，系统 MUST 在内部尝试全部结束后统一汇总成功尝试结果并更新生长任务。

#### Scenario: 全部尝试成功
- **WHEN** 生长任务的所有内部尝试都成功创建果实
- **THEN** 系统 MUST 将任务标记为已完成
- **AND** 系统 MUST 保留全部已创建果实

#### Scenario: 部分尝试成功
- **WHEN** 生长任务至少一个内部尝试成功且至少一个内部尝试失败
- **THEN** 系统 MUST 将任务标记为已完成
- **AND** 系统 MUST 不回滚已成功创建的果实

#### Scenario: 全部尝试失败
- **WHEN** 生长任务没有任何内部尝试成功创建果实
- **THEN** 系统 MUST 将任务标记为失败
- **AND** 系统 MUST 不创建空果实或占位果实

#### Scenario: 并发尝试结束后统一汇总任务结果
- **WHEN** 生长任务的并发内部尝试全部结束
- **THEN** 系统 MUST 基于内部尝试记录汇总成功果实引用
- **AND** 系统 MUST 一次性更新生长任务成功果实集合、最终状态和完成时间
