## Context

果实模块是内容森林枝化生长结果的后端落地点。根据 `docs/design/domain/果实领域模块设计文档.md`，果实由枝化生长交付的果实候选创建，创建成功后成为内容树中的可操作节点，支持用户查看、编辑正文，并在人为物竞天择中切换候选、已选择、已淘汰三类状态。

当前后端已经有 `seed` 与 `generator` 模块，形成了应用服务、领域类型、存储端口、内容访问端口、HTTP 控制器、SQLite/内存适配器的基本实现模式。果实模块应沿用该模式，同时遵守项目架构中“文件为内容本体，数据库为系统事实”的混合存储原则。果实 Markdown 正文存放在内容访问层背后的文件存储；果实身份、选择状态、父节点引用、内容位置、摘要、基因标签等系统事实由数据库维护。

## Goals / Non-Goals

**Goals:**

- 提供从果实候选创建果实的内部应用能力，并保证前端不能直接调用创建果实。
- 保存果实 Markdown 正文，并在数据库中维护果实系统事实。
- 创建果实时必须建立父节点引用，父节点可以是种子节点或果实节点。
- 新果实默认处于候选状态。
- 支持查看果实详情，通过内容本体位置读取 Markdown 正文。
- 支持编辑果实 Markdown 正文，且编辑不改变果实身份、父节点引用、生成来源或选择状态。
- 支持选择果实、淘汰果实、恢复为候选。
- 支持内部按父节点查询子果实，供后续内容树工作区模块使用。
- 支持内部可发布判断，供后续发布验证模块限制只有已选择果实才能发布。
- 补齐 `docs/api/fruit.yaml` 与 `docs/sql/fruit.sql` 顶层契约文档。

**Non-Goals:**

- 不提供面向前端的果实创建接口。
- 不提供全局果实列表页面或全局果实列表 API。
- 不提供删除果实能力。
- 不强制选择/淘汰原因。
- 不调用 Agent，不执行生成器，不处理枝化生长任务。
- 不判断生长锁，不决定果实当前是否能发起枝化生长。
- 不实现发布验证、数据回流、基因汲取或工作区画布布局。

## Decisions

### 决策一：果实创建作为内部应用能力

果实只能由枝化生长模块交付的果实候选创建。HTTP 控制器不暴露创建果实接口给前端，避免用户绕过枝化生长流程凭空创建果实。

替代方案是提供公开创建接口，方便调试和前端直连。该方案会破坏“果实只能由果实候选创建”的领域约束，因此不采用。测试可以直接调用应用服务或使用专用测试装配完成。

### 决策二：果实正文文件化，meta 数据库化

果实 Markdown 正文属于内容本体，通过内容访问层保存到 `data/content/fruits/`。果实状态、父节点引用、内容位置、摘要、基因标签等属于系统事实，由数据库维护，不能写入 Markdown。

替代方案是在 Markdown frontmatter 中保存状态和基因标签。该方案会形成双事实来源，并违反现有架构文档中 Markdown 不保存数据库维护 meta 的约束，因此不采用。

### 决策三：父节点引用由果实模块保存

果实创建时保存 `parentNodeRef` 语义，用于表达果实挂载到哪个种子节点或果实节点下。内容树工作区后续可以读取该关系来组装树视图。

替代方案是由工作区模块单独维护所有边关系。该方案会让果实创建和树挂载拆成两个事实来源，增加创建不完整果实的风险。第一期由果实模块保存父节点引用更符合领域文档。

### 决策四：物竞天择状态保持最小互斥集合

果实选择状态只包含候选、已选择、已淘汰三类。候选即暂时保留观察，不单独设计已保留状态。选择、淘汰、恢复候选都只是状态切换，不删除内容和历史关系。

替代方案是引入更多状态，如已保留、已发布、已反馈。该方案会混合发布验证和数据回流领域语义，因此不采用。已发布、已反馈应由后续领域维护或派生。

### 决策五：摘要和基因标签属于果实 meta，但创建时允许为空

果实摘要和基因标签由枝化生长结果提供，并作为果实 meta 存储。创建果实时 Markdown 正文和父节点引用必须存在；摘要可以为空，基因标签可以为空集合，以降低 Agent 输出不完整导致整个果实无法落地的概率。

替代方案是要求摘要和基因标签必填。该方案看似更完整，但会把枝化生长输出质量问题扩大为果实创建失败，不利于第一期闭环稳定。

### 决策六：按父节点查询子果实作为内部能力

果实模块不提供全局果实列表，但需要提供按父节点查询子果实的内部能力，供内容树工作区模块组装树视图。该能力不是用户意义上的果实列表功能。

替代方案是等内容树模块实现时再追加查询。该方案会导致内容树模块直接理解果实存储细节，不利于模块边界。

## Risks / Trade-offs

- [Risk] 内容文件保存成功但数据库写入失败导致孤儿文件 → Mitigation：创建果实时参考生成器模块模式，在系统事实写入失败时清理已创建的果实 Markdown 文件。
- [Risk] 果实模块保存父节点引用可能与未来内容树模块边界重叠 → Mitigation：果实模块只保存创建时的父节点事实，工作区模块只读取并组织视图，不在第一期支持改父节点。
- [Risk] 不提供公开创建接口导致调试不便 → Mitigation：通过应用服务测试和后续枝化生长模块集成测试验证创建能力。
- [Risk] 摘要和基因标签允许为空会降低展示完整性 → Mitigation：详情展示需要兼容空摘要和空基因标签，后续可通过枝化生长输出质量提升补足。
- [Risk] 状态切换过于自由可能允许已淘汰果实重新选择 → Mitigation：领域状态机明确允许恢复和重新选择，淘汰不是删除，保留反悔空间。
- [Risk] API/SQL 契约与实现脱节 → Mitigation：tasks 中将 `docs/api/fruit.yaml` 与 `docs/sql/fruit.sql` 作为首要任务，并要求实现能回溯到 spec。
