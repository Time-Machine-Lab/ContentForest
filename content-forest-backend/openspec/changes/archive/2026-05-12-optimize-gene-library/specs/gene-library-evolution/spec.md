## ADDED Requirements

### Requirement: 记录基因使用结果
系统 SHALL 支持为正式基因经验记录后续使用结果。每条使用记录 MUST 归属于明确种子和正式基因经验，并 MUST 表达本次使用的结果倾向和证据来源。

#### Scenario: 记录正向使用结果
- **WHEN** 某条基因经验在后续生成或验证中被使用并产生正向结果
- **THEN** 系统 MUST 保存该基因经验的正向使用记录
- **AND** 系统 MUST 将该记录归属于当前种子级基因库

#### Scenario: 记录负向使用结果
- **WHEN** 某条基因经验在后续生成或验证中被使用并产生负向结果
- **THEN** 系统 MUST 保存该基因经验的负向使用记录
- **AND** 系统 MUST 保留该基因经验本身，不得自动归档或删除

#### Scenario: 拒绝跨种子记录
- **WHEN** 用户或系统尝试在一个种子下记录其他种子的基因经验使用结果
- **THEN** 系统 MUST 拒绝该记录
- **AND** 系统 MUST 返回基因经验不属于当前种子的错误

### Requirement: 维护基因表现汇总
系统 SHALL 基于基因使用记录维护正式基因经验的表现汇总。表现汇总 MUST 至少表达使用次数、正向次数、负向次数、中性次数、最近使用时间和轻量评分。

#### Scenario: 使用记录更新表现汇总
- **WHEN** 系统成功保存一条基因使用记录
- **THEN** 系统 MUST 更新对应基因经验的表现汇总
- **AND** 表现汇总 MUST 能被后续查询和 Agent 上下文读取

#### Scenario: 无使用记录的基因仍可查询
- **WHEN** 某条正式基因经验尚无使用记录
- **THEN** 系统 MUST 允许查询该基因经验
- **AND** 系统 MUST 将其表现汇总视为零使用状态

### Requirement: 查询种子级基因库演化摘要
系统 SHALL 支持查询某个种子级基因库的演化摘要。演化摘要 MUST 包含正式基因经验、谱系分布和表现汇总，不得返回其他种子的基因数据。

#### Scenario: 查询当前种子的演化摘要
- **WHEN** 用户查询某个种子的基因库演化摘要
- **THEN** 系统 MUST 返回该种子下正式基因经验的表现摘要
- **AND** 系统 MUST 按轻量评分或最近更新时间提供稳定排序

#### Scenario: 不混入其他种子的基因
- **WHEN** 查询某个种子的基因库演化摘要
- **THEN** 系统 MUST NOT 返回其他种子的基因经验、使用记录或表现汇总

### Requirement: 支持谱系表现回看
系统 SHALL 支持按基因谱系组织表现回看。谱系回看 MUST 基于同一种子内的正式基因经验和使用记录形成，并 MUST 允许多个谱系长期并存。

#### Scenario: 多谱系并存回看
- **WHEN** 同一种子下存在多个基因谱系
- **THEN** 系统 MUST 能分别汇总各谱系的使用次数和结果倾向
- **AND** 系统 MUST NOT 因某个谱系表现较弱而自动禁用该谱系

#### Scenario: 空谱系归入未分组
- **WHEN** 某条基因经验没有明确谱系
- **THEN** 系统 MUST 在演化摘要中将其纳入未分组谱系
- **AND** 系统 MUST 保持该基因经验仍可被引用

### Requirement: 基因库演化保持轻量边界
系统 SHALL 在第一期以轻量统计实现基因库演化。系统 MUST NOT 因基因库演化引入独立实验平台、自动适应度服务、消息队列或新的必需外部服务。

#### Scenario: 不引入重型基础设施
- **WHEN** 系统实现基因库持续跟进、评优和迭代能力
- **THEN** 系统 MUST 使用现有后端应用服务、存储端口、SQLite 和 AgentPort 边界完成
- **AND** 系统 MUST NOT 要求部署新的外部基础设施

#### Scenario: 评分不作为绝对事实
- **WHEN** 系统返回基因表现评分
- **THEN** 评分 MUST 仅作为排序和回看辅助信息
- **AND** 系统 MUST NOT 基于评分自动确认、删除、归档或改写基因经验
