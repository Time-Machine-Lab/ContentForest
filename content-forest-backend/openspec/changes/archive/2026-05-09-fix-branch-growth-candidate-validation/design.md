## Context

当前枝化生长 Agent Skill 会把生成器 payload 再封装为候选果实结构，并在交给 Growth 领域前执行本地校验。真实 LLM 输出中，`usedResourceRefs` 容易出现字符串数组、字段缺失或格式漂移，导致候选果实校验失败并使枝化生长尝试失败。

内容森林 Agent 架构要求：Agent 只返回候选结果，不直接落地果实；文件内容读取由受控 Tool 完成；授权范围由任务上下文限定；外部生成器只负责 payload，不负责果实 meta。因此本次修复只调整枝化生长 Agent Skill 的结构化封装、校验归一化与自检修复行为，不变更 API、数据库或 Growth/Fruit 领域落地规则。

## Goals / Non-Goals

**Goals:**

- 提升真实 LLM 输出下候选果实结构化校验的成功率。
- 明确 `usedResourceRefs` 的标准结构，并让提示词和修复提示都围绕该结构收敛。
- 在不放松授权边界的前提下，兼容可确定映射的字符串资源引用。
- 保证校验失败原因能传递给修复模型和运行 Trace，方便定位。
- 增加测试覆盖，防止资源引用校验回退为不安全的宽松接受。

**Non-Goals:**

- 不新增或修改 HTTP API。
- 不新增或修改数据库表结构。
- 不改变 Growth 任务状态流转、果实创建流程或内容文件写入流程。
- 不让 Agent 获得任何写入数据库、写入文件或直接创建果实的能力。
- 不实现营养库或基因库 Tool 的读取细节。

## Decisions

### Decision 1: 以标准对象数组作为资源引用的唯一落地结构

候选果实最终通过校验后，`usedResourceRefs` 必须统一为对象数组，每个引用表达资源类型与资源标识。这样 Growth 领域和后续 Trace 不需要理解 LLM 的多种临时写法。

Alternative considered: 允许字符串数组作为系统内部结构继续向下游传递。该方案会把 LLM 输出不确定性扩散到 Growth/Fruit 领域，违背“把不确定性交给生成器，把确定性交给枝化生长，把系统事实交给内容森林”的边界。

### Decision 2: 字符串资源引用只在授权集合内可唯一识别时归一化

当 LLM 返回字符串引用时，校验器可以根据本次任务授权资源集合进行归一化。只有字符串能唯一匹配某个授权资源时，才转换为标准对象引用；空字符串、未授权引用、无法识别引用或多类型歧义引用都必须失败。

Alternative considered: 对所有字符串引用默认丢弃。该方案最保守，但会让模型轻微格式漂移直接导致可用结果失败。当前方案在授权边界内做可解释修复，更适合真实 LLM 运行。

### Decision 3: Prompt 与 Repair Prompt 同时强化

初次结构化封装提示必须给出明确 JSON 结构要求，尤其是资源引用对象数组示例。自检修复提示必须包含真实校验错误、原始输出和原始 payload，并要求只修复结构与缺失字段。

Alternative considered: 只修改校验器兼容字符串引用。该方案能缓解当前错误，但无法减少模型持续输出错误结构的概率，也不利于未来扩展更多结构字段。

### Decision 4: 校验仍是 AgentPort 返回前的强制门禁

归一化只用于把模型输出收敛为系统内部结构，不代表放弃严格校验。候选果实仍必须满足 Markdown 非空、摘要可用、基因标签可用、资源引用授权、禁止伪造系统事实等规则。

Alternative considered: 校验失败时返回半成品给 Growth 领域。该方案会把 Agent 层职责泄漏给业务领域，并使用户看到不可落地的候选果实。

## Risks / Trade-offs

- [Risk] 字符串引用匹配规则过宽可能误接收未授权资源 → Mitigation: 仅基于本次任务授权资源集合匹配，且必须唯一匹配。
- [Risk] 授权集合中存在相同资源标识但类型不同会产生歧义 → Mitigation: 将歧义视为校验失败，要求模型返回完整对象结构。
- [Risk] Prompt 强化仍不能保证所有模型稳定遵守格式 → Mitigation: 保留本地 Schema 校验与有限次数 Repair 流程。
- [Risk] 过度宽容会掩盖生成器或 Agent 的真实输出问题 → Mitigation: Trace 保留归一化、校验失败和修复失败原因，测试覆盖非法引用失败。

## Migration Plan

本次变更不涉及数据迁移和接口迁移。发布后，新校验逻辑只影响后续枝化生长 Agent Skill 的候选果实输出处理。若出现异常，可回滚到旧版本代码；已生成果实与已有任务数据不需要处理。
