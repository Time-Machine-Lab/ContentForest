## ADDED Requirements

### Requirement: 定义可迭代内容进化算法模型
系统 SHALL 定义内容进化算法模型作为枝化生长和基因汲取之间的轻量策略语义。该模型 MUST 表达算法版本、输入数据源、策略产物、基因假设和反馈调节边界，并 MUST 允许后续版本替换策略实现而不改变生成器、果实、营养库和基因库的领域归属。

#### Scenario: 使用算法版本标识策略模型
- **WHEN** 系统执行枝化生长或基因汲取 Agent 任务
- **THEN** 系统 MUST 能在任务上下文、Trace 或日志中表达当前使用的内容进化算法模型版本
- **AND** 该版本标识 MUST 不要求改变公开 API 或数据库结构

#### Scenario: 后续替换策略模型
- **WHEN** 后续版本需要优化资料选择、基因继承、变异或反馈调节规则
- **THEN** 系统 MUST 允许替换或升级策略模型版本
- **AND** 生成器 Skill、果实领域、营养库领域和基因库领域的核心边界 MUST 保持不变

### Requirement: 组织多源数据为生长策略
系统 SHALL 将来源节点、用户输入、生成器方法论、营养资料、基因经验和历史反馈组织为生长策略。生长策略 MUST 说明本轮生成目标、采用证据、继承或规避的基因、探索方向和成功判据，而不是把所有资料无差别拼接为单一上下文。

#### Scenario: 从多源上下文形成策略
- **WHEN** 枝化生长 Agent 已读取本次授权的来源节点、生成器、营养和基因上下文
- **THEN** 系统 MUST 形成可用于指导内容生成的生长策略
- **AND** 生长策略 MUST 能说明每类上下文对本次生成的作用

#### Scenario: 不使用未授权资料
- **WHEN** 某个资料或基因不在本次任务授权范围内
- **THEN** 系统 MUST 不将其纳入生长策略
- **AND** 系统 MUST 不把该资料作为候选果实的使用资源引用返回

### Requirement: 支持批量果实差异化探索
系统 SHALL 在一次枝化生长批次中支持 attempt 级差异化探索。每个果实生成尝试 MUST 拥有可区分的策略方向，系统 MUST 避免仅以同一上下文重复采样来替代多样性探索。

#### Scenario: 为多个 attempt 分配探索方向
- **WHEN** 用户请求一次枝化生长生成多个果实
- **THEN** 系统 MUST 为每个果实生成尝试分配或形成不同的探索方向
- **AND** 每个探索方向 MUST 能回溯到本次生长策略

#### Scenario: 单个 attempt 仍可使用策略
- **WHEN** 用户请求一次枝化生长只生成一个果实
- **THEN** 系统 MUST 仍然可以形成单个 attempt 的生长策略
- **AND** 系统 MUST 不要求额外的批量对比信息才能运行

### Requirement: 将基因表达为可验证假设
系统 SHALL 将基因表达为可验证的内容表达特征假设。基因假设 MUST 能表达特征内容、作用方向、适用生态位、证据来源和下一轮使用建议。

#### Scenario: 表达正向基因假设
- **WHEN** 系统从选择或优秀反馈中汲取经验
- **THEN** 系统 MUST 能将其表达为值得继承、强化、组合或变异的正向基因假设
- **AND** 该假设 MUST 包含可追溯证据来源

#### Scenario: 表达反向基因假设
- **WHEN** 系统从淘汰或低表现反馈中汲取经验
- **THEN** 系统 MUST 能将其表达为应该规避、抑制或谨慎使用的反向基因假设
- **AND** 该假设 MUST 包含适用边界，避免被错误地全局禁止

### Requirement: 保持算法模型轻量落地
系统 SHALL 在第一期以轻量方式落地内容进化策略模型。系统 MUST NOT 因本能力引入向量数据库、独立实验平台、消息队列、自动发布系统或新的必需外部服务。

#### Scenario: 不新增重型基础设施
- **WHEN** 实现内容进化策略模型第一版
- **THEN** 系统 MUST 使用现有 Agent Runtime、Tool、Skill、Trace、日志和领域服务边界完成
- **AND** 系统 MUST 不要求部署新的外部基础设施

#### Scenario: 不改变系统事实归属
- **WHEN** 策略模型生成生长策略、证据卡片或基因假设
- **THEN** 系统 MUST 保持系统事实仍由对应领域服务维护
- **AND** Agent MUST NOT 直接写数据库、写 Markdown 文件或确认基因经验
