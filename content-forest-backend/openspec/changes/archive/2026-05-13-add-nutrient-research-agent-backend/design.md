## Context

第二期营养工作台的中间区域是 Agent 联网研究对话区。用户围绕种子或营养卡片提问，Agent 必须能联网搜索并产出可沉淀营养块。该能力不同于枝化生长：它不创建果实，不做物竞天择，只服务于补充种子专属营养。

## Goals / Non-Goals

**Goals:**
- 新增营养研究 Agent 能力，支持联网搜索。
- 支持会话式交流和结构化可沉淀营养块。
- 让 Agent 输出可被营养领域消费，而不是直接写库。
- 记录研究 Trace，便于排查搜索和生成问题。

**Non-Goals:**
- 不实现自动爬虫批量采集。
- 不展示复杂来源可信度管理。
- 不自动沉淀营养。
- 不把研究结果保存到公共营养库。

## Decisions

### Decision 1: 新增 `nutrient_research` 任务类型

营养研究的输入、工具和输出都不同于 growth/gene/seed_brief，因此新增独立任务类型比复用 growth 更清晰。

### Decision 2: 输出分为普通消息与可沉淀营养块

普通消息用于解释、追问和协作；可沉淀营养块用于生成未沉淀营养卡片。结构化输出可以让前端特殊展示并让后端校验。

### Decision 3: 联网搜索必须受控

搜索 Tool 应由 Agent Runtime 注册，记录查询词、结果摘要和失败原因。Skill 不应获得任意本地文件访问权限，也不应直接写数据库。

### Decision 4: 会话归属于种子，可选绑定营养卡片

用户可以在工作台直接研究，也可以点开某张卡片继续优化。因此会话必须归属 seedId，并可选绑定 nutrientCardId。

## Risks / Trade-offs

- [Risk] 联网搜索结果质量不稳定 → 限制输出为候选营养，不自动沉淀。
- [Risk] 搜索依赖外部服务失败 → 任务和消息需记录失败原因，允许用户重试。
- [Risk] LLM 将来源内容当系统指令 → Skill prompt 必须把搜索结果视为参考资料。
