## Why

内容森林第一期需要先建立“灵感种子”这一根节点能力，否则后续工作区、枝化生长、果实挂载都缺少稳定的业务起点。该变更依据 `docs/design/domain/种子领域模块设计文档.md`、`docs/design/内容森林架构设计文档.md` 和 `docs/内容森林第一期开发规划文档.md`，实现种子作为可编辑、可归档、可作为内容树根的核心领域模块。

## What Changes

- 新增种子模块后端能力：创建种子、查看未归档种子列表、查看已归档种子列表、查看种子详情、编辑种子标题、编辑种子 Markdown 内容、归档种子、回档种子。
- 新增种子作为内容树根节点和枝化生长来源的领域语义：未归档种子可作为生长源，已归档种子及其工作区只读，不可继续发起枝化生长。
- 新增种子 Markdown 内容本体存储能力：种子正文保存为 Markdown 内容本体，数据库只维护系统事实和内容本体位置。
- 新增种子模块所需的 API 与 SQL 顶层设计文档要求：在实现前必须创建或更新 `docs/api/` 下对应 Controller 的 OpenAPI 文档，以及 `docs/sql/` 下对应数据库表结构文档。
- 不引入删除能力、草稿态、标签、分类、平台方向、搜索、高级排序或种子专属营养库维护。

## Capabilities

### New Capabilities

- `seed-management`: 定义种子的创建、查看、编辑、归档、回档、列表分类、内容本体边界和生长源可用性规则。

### Modified Capabilities

无。

## Impact

- 影响后端业务模块：新增 `seed` 领域模块及其应用服务能力。
- 影响内容访问层：需要支持种子 Markdown 内容本体的保存与读取。
- 影响存储层：需要维护种子系统事实、归档状态、根节点关系和内容本体位置；具体结构必须先落到 `docs/sql/*.sql`。
- 影响 API 契约：需要提供种子创建、列表、详情、编辑、归档、回档等应用接口；具体契约必须先落到 `docs/api/*.yaml`，并遵守“一份 `.yaml` 对应一个 Controller 文件”。
- 影响后续模块：枝化生长模块可依赖种子模块判断未归档种子是否可作为生长源；工作区模块可依赖种子作为内容树根节点。
