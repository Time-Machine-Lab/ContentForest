## Why

内容森林已经具备种子与生成器基础能力，但枝化生长产生的内容结果还没有后端领域模块承接。果实模块需要成为枝化生长结果的落地点，负责把候选内容保存为可查看、可选择、可淘汰、可继续参与后续发布验证和数据回流的果实对象。

本变更基于 `docs/design/domain/果实领域模块设计文档.md`，优先实现第一期后端闭环所需的果实领域能力，不扩展到工作区画布、发布验证、数据回流、Agent 调用或生长锁判断。

## What Changes

- 新增果实后端领域模块，包含领域类型、应用服务、存储端口、内容访问端口和 HTTP 控制器。
- 支持由后端内部从果实候选创建果实；果实创建不允许前端直接调用。
- 支持保存果实 Markdown 正文，并将果实状态、父节点引用、内容位置、摘要、基因标签等 meta 作为系统事实维护。
- 支持查看果实详情、编辑果实正文、选择果实、淘汰果实、恢复为候选。
- 支持内部按父节点查询子果实，供后续内容树工作区组装树视图使用；不提供面向用户的全局果实列表功能。
- 支持内部判断果实是否可发布，供发布验证模块限制“只有已选择果实才能发布”。
- 补充果实相关顶层契约文档：`docs/api/fruit.yaml` 与 `docs/sql/fruit.sql`。
- 不提供删除果实能力；淘汰只是状态变化，不移除内容和历史关系。

## Capabilities

### New Capabilities

- `fruit-management`: 果实创建落地、果实详情读取、正文编辑、物竞天择状态管理、父节点子果实查询与可发布判断。

### Modified Capabilities

- None.

## Impact

- 新增后端模块：`src/modules/fruit/`。
- 新增存储端口与适配器：`src/storage/ports/fruit-storage-port.ts`、内存适配器、SQLite 适配器。
- 新增内容访问端口与本地文件适配器，用于读写 `data/content/fruits/` 下的 Markdown 正文。
- 新增 HTTP 控制器：`src/interface/http/fruit-controller.ts`，但不暴露“创建果实”给前端。
- 更新应用装配与运行时目录初始化，确保果实内容目录可用。
- 新增测试覆盖果实服务、内容访问、存储适配和核心 HTTP 行为。
- 新增或更新顶层契约文档：`docs/api/fruit.yaml`、`docs/sql/fruit.sql`。
