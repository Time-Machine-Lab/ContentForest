## Why

内容森林已经具备发布验证模块，可以为已选择果实创建发布记录；但数据回流领域仍停留在设计文档阶段，真实发布后的表现数据无法回到系统，也无法为后续基因汲取提供反馈上下文。现在需要补齐“发布记录 -> 监控器 -> 反馈快照”的后端闭环，让第一期物竞天择流程具备可追踪的数据回流入口。

## What Changes

- 新增数据回流后端模块，围绕发布记录挂载监控器并记录反馈快照。
- 新增人为监控器能力：第一期只记录用户手动查看外部平台后回填的数据，不调用任何外部平台 API。
- 新增监控器挂载能力：一条发布记录只能挂载一个监控器；挂载前必须校验发布记录存在。
- 新增反馈快照创建能力：基于已挂载监控器的发布记录追加多次快照，记录自由结构的表现数据和用户观察。
- 新增反馈快照编辑能力：允许修正表现数据和用户观察，但不允许删除快照。
- 新增反馈历史查询能力：支持按发布记录查看监控器挂载状态和全部反馈快照。
- 新增供基因汲取读取的反馈上下文能力，使 `publication` 和 `feedback` 证据能够追溯到发布记录及快照历史。
- 新增 `docs/api/feedback.yaml` 与 `docs/sql/feedback.sql` 顶层契约；一份 API 契约对应一个 Feedback Controller，一张反馈相关表结构契约对应一个 SQL 文档。
- 不新增监控器管理页面、不做自动抓取、不做固定平台指标枚举、不判断内容成败、不修改果实状态、不修改发布记录本身。

## Capabilities

### New Capabilities

- `data-feedback`: 管理发布记录上的人为监控器挂载、反馈快照创建/编辑/查询，并为基因汲取提供数据回流上下文。

### Modified Capabilities

- `gene-extraction`: 扩展基因汲取可读取的反馈证据上下文，使 `feedback` 类型证据能够基于数据回流快照形成输入材料。

## Impact

- 影响顶层契约：新增 `docs/api/feedback.yaml` 和 `docs/sql/feedback.sql`。
- 影响后端模块层：新增 `src/modules/feedback/`，包含领域类型、应用服务、模块出口。
- 影响存储层：新增反馈存储端口、内存适配器和 SQLite 适配器。
- 影响接口层：新增 Feedback Controller，并在 HTTP 路由中装配反馈相关接口。
- 影响应用装配：需要注入发布记录存在性校验依赖，并注册反馈服务、存储适配器和控制器。
- 影响基因汲取工具：读取证据上下文时需要能读取反馈快照历史。
- 影响测试：覆盖监控器挂载、重复挂载拒绝、无发布记录拒绝、快照创建/编辑/查询、禁止删除、反馈证据上下文读取等核心路径。
