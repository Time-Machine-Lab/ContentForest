## Why

当前种子 Markdown 已通过内容访问层保存，但后端启动时尚未统一初始化运行时文件系统根目录，后续营养库、生成器、果实等模块容易各自拼接路径和创建目录，破坏 `docs/design/内容森林架构设计文档.md` 中“内容本体文件化、系统事实数据库化”的边界。

本变更建立统一的 Runtime Filesystem Bootstrap：后端启动时创建可配置的内容根目录及标准子目录，并要求数据库中的内容位置统一保存相对路径，避免本机绝对路径污染系统事实。

## What Changes

- 新增后端启动阶段的运行时文件系统初始化能力。
- 新增统一内容根目录约定，默认用于保存种子、果实、营养、生成器、附件和临时文件等内容本体。
- 明确 `contentLocation` 等内容位置必须保存相对路径，不得保存本机绝对路径。
- 修改架构设计文档，补充运行时文件系统根目录、目录骨架和 Bootstrap 职责。
- 修改后端开发规范文档，补充运行时目录初始化、路径存储和业务模块路径边界规则。
- 调整本地内容访问适配器，使其基于统一内容根目录解析相对路径。
- 不新增或修改 API 契约，不新增或修改数据库表结构。

## Capabilities

### New Capabilities

- `runtime-filesystem-bootstrap`: 定义后端启动时创建运行时文件系统根目录、标准内容子目录、相对内容位置和内容访问适配器装配的行为。

### Modified Capabilities

无。

## Impact

- 影响 `src/app/config`：需要提供运行时内容根目录和数据库路径等配置读取能力。
- 影响 `src/app/bootstrap`：需要在应用启动装配前初始化运行时文件系统目录。
- 影响 `src/content-access/adapters`：本地文件适配器应基于内容根目录解析相对路径。
- 影响 `docs/design/内容森林架构设计文档.md`：补充运行时文件系统设计。
- 影响 `docs/spec/后端开发规范文档.md`：补充 Bootstrap 初始化和相对路径规则。
- 影响后续种子、果实、营养库、生成器等模块：内容位置统一写入相对路径，真实绝对路径只存在于内容访问适配器内部。
