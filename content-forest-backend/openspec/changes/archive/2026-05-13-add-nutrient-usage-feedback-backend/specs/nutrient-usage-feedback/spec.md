## ADDED Requirements

### Requirement: 记录营养使用关系
系统 SHALL 记录枝化生长中使用的正式营养和未沉淀营养卡片，并关联到生长任务、生成尝试和生成果实。

#### Scenario: 生长使用正式营养
- **WHEN** 枝化生长成功创建果实且本次使用了正式营养
- **THEN** 系统 MUST 记录该营养与生长任务、果实之间的使用关系

#### Scenario: 生长使用未沉淀营养
- **WHEN** 枝化生长成功创建果实且本次使用了未沉淀营养卡片
- **THEN** 系统 MUST 记录该卡片与生长任务、果实之间的临时使用关系
- **AND** 系统 MUST 不因此将该卡片标记为已沉淀

### Requirement: 汇总营养使用效果
系统 SHALL 能基于营养使用关系汇总后续效果摘要。摘要 MUST 可以包含相关果实数量、已选择数量、已淘汰数量、发布记录数量和反馈快照摘要。

#### Scenario: 查看营养效果摘要
- **WHEN** 用户查看营养卡片详情
- **THEN** 系统 MUST 返回该营养影响过的生长和果实摘要
- **AND** 系统 MUST 返回相关选择、淘汰、发布和反馈摘要

#### Scenario: 不推断因果结论
- **WHEN** 系统返回营养使用效果摘要
- **THEN** 系统 MUST NOT 自动声明该营养导致果实成功或失败

### Requirement: 提供新鲜度提醒
系统 SHALL 为营养卡片维护最近研究时间、最近更新时间和最近引用时间。系统 MUST 能识别长期未更新或长期未引用的营养，并返回刷新提醒。

#### Scenario: 营养长期未更新
- **WHEN** 营养卡片超过配置周期未更新
- **THEN** 系统 SHOULD 在营养卡片摘要中返回建议重新研究的提醒

#### Scenario: 营养被引用后更新时间
- **WHEN** 营养卡片被枝化生长引用
- **THEN** 系统 MUST 更新最近引用时间或记录等价使用事实

### Requirement: 支持轻量去重与合并
系统 SHALL 在创建新营养卡片或可沉淀营养块时支持相似营养提示，并允许用户确认合并到已有卡片。

#### Scenario: 提示相似营养
- **WHEN** 系统生成新的可沉淀营养块
- **THEN** 系统 SHOULD 返回可能相似的已有营养卡片摘要

#### Scenario: 合并到已有卡片
- **WHEN** 用户确认将新营养内容合并到已有卡片
- **THEN** 系统 MUST 更新目标卡片内容或追加整理记录
- **AND** 系统 MUST 保留合并来源的可追溯关系
