## Context

用户希望营养库保持新潮和新鲜，且减少手动总结平台规律的工作。第一版不做复杂评分，但需要把营养引用与后续结果串起来，让用户看到哪些营养被用过、是否值得刷新或合并。

## Goals / Non-Goals

**Goals:**
- 记录营养在枝化生长中的使用。
- 关联生成果实及其后续状态和反馈摘要。
- 提供新鲜度提醒所需字段。
- 支持轻量去重和合并。

**Non-Goals:**
- 不自动判断营养好坏。
- 不自动归档或删除营养。
- 不做向量相似检索。
- 不展示复杂来源可信度。

## Decisions

### Decision 1: 引用记录由生长结算写入

枝化生长任务知道本次授权和实际使用的营养资源，也知道最终创建了哪些果实。由生长结算阶段写入引用关系最稳定。

### Decision 2: 效果回流使用摘要，不计算统一评分

第一版只汇总选择、淘汰、发布和反馈数量/摘要，不把它变成自动质量分，避免误导用户。

### Decision 3: 新鲜度以提醒为主

记录最近研究、最近更新、最近引用时间。是否刷新由用户决定，不自动联网刷新。

### Decision 4: 去重合并先做轻量规则

第一版基于标题、主题、关联卡片和 Agent 提示进行相似提醒，不引入向量库。

## Risks / Trade-offs

- [Risk] 引用记录不代表营养真实有效 → 明确是效果摘要，不是因果结论。
- [Risk] 合并可能覆盖用户整理内容 → 合并必须由用户确认，并保留原卡片历史。
- [Risk] 数据过多影响界面 → 后端提供摘要，详情按需查询。
