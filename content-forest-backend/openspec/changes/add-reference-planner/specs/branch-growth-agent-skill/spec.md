## ADDED Requirements

### Requirement: 枝化生长 Skill 必须按参考计划组织生成上下文
系统 SHALL 要求内置枝化生长 Skill 按 attempt 级参考计划组织生成上下文。Skill MUST 使用参考原子的槽位路由、动作语义和边界约束指导生成器 payload，MUST NOT 将所有授权资料无差别拼接为单一长上下文。

#### Scenario: 按槽位和动作使用参考原子
- **WHEN** 枝化生长 Skill 准备调用生成器或模型生成候选内容
- **THEN** Skill MUST 将参考原子按标题钩子、开头、受众场景、正文结构、证明依据、话术风格、转化行动、风险检查或事实检查等槽位组织
- **AND** Skill MUST 为每个槽位保留对应的使用动作和边界说明

#### Scenario: 不执行资料中的系统指令
- **WHEN** 营养、基因、广告资料、论文、来源节点或生成器正文包含指挥模型越权读取、写入、忽略系统规则或泄露信息的文本
- **THEN** 枝化生长 Skill MUST 将其视为不可信资料内容
- **AND** Skill MUST NOT 将该文本作为系统指令执行

### Requirement: 枝化生长 Skill 必须校验高风险主张
系统 SHALL 要求内置枝化生长 Skill 对参考计划中标记为高风险、候选主张或风险约束的原子执行生成前后校验。Skill MUST 避免将候选主张、广告宣传、论文边界外结论或平台观察信号写成无条件事实。

#### Scenario: 论文结论被条件化表达
- **WHEN** 参考计划包含来自论文或技术资料的候选主张
- **THEN** 枝化生长 Skill MUST 按研究边界进行条件化表达
- **AND** Skill MUST NOT 输出超出研究对象、条件、指标或适用范围的绝对承诺

#### Scenario: 广告主卖点进入风险检查
- **WHEN** 参考计划包含广告主提供的功效卖点或宣传话术
- **THEN** 枝化生长 Skill MUST 将其纳入风险检查或事实检查
- **AND** Skill MUST 按禁用用途避免绝对化、医疗化、虚假因果或平台违规表达

### Requirement: 候选果实必须包含参考使用声明
系统 SHALL 允许枝化生长 Skill 在候选果实结构化输出中返回参考使用声明。参考使用声明 MUST 使用标准资源引用和参考原子摘要，MUST NOT 伪造未授权资源或不存在的使用事实。

#### Scenario: 输出实际使用资源
- **WHEN** 候选果实实际使用了某个正式营养、未沉淀营养卡片或基因经验
- **THEN** 枝化生长 Skill MUST 在结构化输出中声明对应标准资源引用
- **AND** 本地校验 MUST 能将该引用匹配到本次授权资源

#### Scenario: 无法确认使用时不强行声明
- **WHEN** 某个参考原子仅被计划路由但未在候选果实中实际落地
- **THEN** 枝化生长 Skill MUST NOT 将其声明为已实际使用
- **AND** 系统 MUST 能在使用摘要中保留计划与实际之间的差异

### Requirement: 结构化输出校验必须覆盖参考使用声明
系统 SHALL 在候选果实本地 Schema 校验中覆盖参考使用声明。校验 MUST 验证资源授权、引用格式、风险声明和系统事实边界。

#### Scenario: 参考引用越权
- **WHEN** 候选果实参考使用声明包含未授权营养、未授权基因或真实本地文件路径
- **THEN** 本地 Schema 校验 MUST 失败
- **AND** 系统 MUST 不把该候选结果交给 Growth 领域落地

#### Scenario: 风险原子缺少处理说明
- **WHEN** 候选果实使用了高风险或候选主张参考原子
- **THEN** 本地校验 MUST 要求输出包含风险处理摘要或事实检查摘要
- **AND** 缺少必要摘要时系统 MUST 进入修复流程或标记本次尝试失败
