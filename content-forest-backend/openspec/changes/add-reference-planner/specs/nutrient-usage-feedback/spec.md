## ADDED Requirements

### Requirement: 营养使用反馈必须区分提供、计划和实际使用
系统 SHALL 在枝化生长营养使用反馈中区分提供的营养、计划参考的营养原子和实际使用的营养引用。系统 MUST 不把被授权提供的营养自动等同于已被候选果实实际采用。

#### Scenario: 正式营养被提供但未计划使用
- **WHEN** 某个正式营养在本次枝化生长授权范围内
- **AND** 参考计划未将其路由到任何 attempt
- **THEN** 系统 MUST 能将其记录为已提供但未计划使用
- **AND** 系统 MUST NOT 将其计入实际使用关系

#### Scenario: 计划参考但未实际落地
- **WHEN** 某个营养原子被参考计划路由到 attempt
- **AND** 候选果实结构化输出未声明或无法校验其实际使用
- **THEN** 系统 MUST 能将其记录为计划参考但未确认实际使用
- **AND** 系统 MUST NOT 将其伪造成已实际使用的营养效果证据

#### Scenario: 未沉淀营养实际使用
- **WHEN** 候选果实通过校验并实际使用了未沉淀营养卡片
- **THEN** 系统 MUST 记录该卡片与生长任务、生成尝试和果实之间的临时使用关系
- **AND** 系统 MUST 不因此将该卡片标记为已沉淀

### Requirement: 营养使用摘要必须表达参考方式而非伪精确权重
系统 SHALL 在营养使用摘要中表达营养被如何参考，包括参考原子类型、目标内容槽位、使用动作、证据强度、风险边界和实际使用状态。系统 MUST NOT 将未校准的模型注意力数值包装为精确权重。

#### Scenario: 返回营养参考方式
- **WHEN** 用户或系统查看某次生长的营养使用摘要
- **THEN** 系统 MUST 能返回每个相关营养的提供状态、计划状态、实际使用状态和参考方式摘要
- **AND** 参考方式 MUST 能说明其用于事实落地、结构塑造、话术风格、证明依据、转化行动、风险检查或规避建议中的哪一类

#### Scenario: 不返回伪精确百分比
- **WHEN** 系统无法基于可校准算法证明某个营养的数值影响比例
- **THEN** 系统 MUST NOT 返回类似精确百分比的权重结论
- **AND** 系统 MUST 使用定性优先语义、槽位路由和使用状态解释其参考程度

### Requirement: 营养使用反馈不得声明因果效果
系统 SHALL 保持营养使用反馈的非因果边界。计划参考、实际使用、果实被选择、果实被淘汰、发布表现和反馈摘要 MUST 仅作为关联摘要展示，系统 MUST NOT 自动声明某个营养导致果实成功或失败。

#### Scenario: 查看营养后续表现
- **WHEN** 用户查看某个营养影响过的生长和果实摘要
- **THEN** 系统 MUST 返回相关提供、计划、实际使用、选择、淘汰、发布和反馈摘要
- **AND** 系统 MUST 明确保持这些信息为关联摘要
- **AND** 系统 MUST NOT 自动输出因果归因结论
