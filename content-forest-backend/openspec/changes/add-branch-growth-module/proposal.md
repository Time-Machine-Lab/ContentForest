## Why

内容森林已经具备种子、生成器、Agent Core 与果实落地能力，但还缺少把“从节点输入 → Agent 生成 → 果实挂载 → 状态回传”串起来的枝化生长领域模块。该变更用于实现第一期工作区最小生长闭环，让用户可以从种子或果实发起枝化生长，并把生成结果稳定挂回内容树。

## What Changes

- 新增枝化生长后端领域模块，覆盖生长任务创建、来源节点校验、资源授权校验、生长锁、状态流转、失败输入恢复和重试。
- 支持从种子或果实作为来源节点发起枝化生长；生成器必选，果实数量默认 3，最大 6。
- 支持一次生长任务内部执行多个果实生成尝试；只要至少一个果实成功创建并挂载，任务即视为已完成；若没有任何果实成功，则任务失败。
- 支持通过 AgentPort 调用枝化生长 Agent 能力；传递任务上下文与授权包，由 Agent 通过 Tool 在授权范围内读取来源内容、生成器、营养与基因。
- 支持将 Agent 返回的候选果实交付给果实模块内部创建能力，并挂载到来源节点下。
- 支持查询生长任务状态，供前端轮询；轮询机制属于前端交互策略，后端只提供状态查询能力。
- 支持查询节点当前是否正在生长，供工作区视图派生节点展示状态；该状态来自进行中的生长任务或生长锁，不属于果实领域。
- 新增或更新顶层契约文档：`docs/api/growth.yaml` 与 `docs/sql/growth.sql`。
- 不实现突变概率、杂交、高级生长策略、并发批量调度、自动选择生成器或复杂任务系统。

## Capabilities

### New Capabilities

- `branch-growth`: 枝化生长任务创建、来源校验、生长锁、AgentPort 调用、候选果实交付、任务状态查询、失败输入恢复与重试。

### Modified Capabilities

无。

## Impact

- 新增后端模块：`src/modules/growth/`。
- 新增存储端口与适配器：用于维护生长任务、生长锁、失败输入和尝试记录等系统事实。
- 新增 HTTP Controller 与 OpenAPI 契约：支持发起生长、重试生长、查询任务状态、查询节点生长状态等应用能力。
- 新增 SQL 契约：维护生长任务与生长尝试系统事实；Markdown 不保存生长任务 meta。
- 依赖既有 `seed`、`fruit`、`generator`、`agent` 模块能力；不直接读写文件、不直接调用 LLM SDK。
- 影响工作区前端后续对接：前端提交生长后获得任务标识，并轮询任务状态，完成后刷新工作区树。
- 新增测试覆盖生长任务状态、锁定释放、数量上限、失败重试、AgentPort 调用、果实创建交付和部分成功不回滚。
