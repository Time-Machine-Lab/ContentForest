## Context

二期规划要求“缺口发现机制必须做”。缺口不是正式营养，也不是 Agent 研究结果，而是系统提示用户“这里可能值得补营养”。它应进入营养工作台右侧建议列表，由用户采纳或忽略。

## Goals / Non-Goals

**Goals:**
- 维护种子级营养汲取建议。
- 从多个轻量信号生成建议。
- 支持采纳建议创建未沉淀营养卡片。
- 支持忽略建议。

**Non-Goals:**
- 不自动发起联网研究。
- 不自动创建正式营养。
- 不做复杂推荐排序模型。

## Decisions

### Decision 1: 建议归属于种子

营养卡片只能保存到种子专属营养库，建议也应绑定 seedId，避免形成全局情报工作区。

### Decision 2: 缺口信号保持轻量

第一版不做复杂算法。建议来源包括种子主简报证据缺口、用户输入中的平台方向、未引用营养的生长请求、淘汰果实和低质量生成反馈等。

### Decision 3: 采纳建议只创建未沉淀卡片

采纳不是沉淀。采纳会创建未沉淀卡片，并把建议内容作为后续 Agent 研究输入；用户仍需继续研究或手动确认后才沉淀。

## Risks / Trade-offs

- [Risk] 建议过多打扰用户 → 支持忽略状态，工作区只展示待处理建议。
- [Risk] 建议质量不稳定 → 建议只作为入口，不自动研究、不自动沉淀。
- [Risk] 多信号重复建议 → 与去重合并能力配合，先做标题和来源级去重。
