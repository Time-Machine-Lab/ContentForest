## Purpose

定义工作区快照聚合、内容树节点与边、生长状态、失败输入提示、可引用资源和无私有持久化状态规则，确保工作区只聚合领域数据而不保存前端布局状态。

## Requirements

### Requirement: 工作区快照读取

系统 MUST 提供工作区快照读取能力，用于根据指定种子聚合工作区首屏加载所需的页面数据。该能力的接口契约 MUST 落到 `docs/api/workspace.yaml`，并由单一工作区 Controller 对应。

#### Scenario: 读取未归档种子的工作区快照

- **WHEN** 前端请求某个未归档种子的工作区快照
- **THEN** 系统返回该种子的工作区只读状态、内容树节点摘要、节点父子关系、节点生长状态和工作区可引用资源摘要

#### Scenario: 读取归档种子的工作区快照

- **WHEN** 前端请求某个已归档种子的工作区快照
- **THEN** 系统返回该种子的工作区快照
- **AND** 系统明确标记该工作区为只读，供前端禁止新的枝化生长入口

#### Scenario: 种子不存在

- **WHEN** 前端请求不存在的种子工作区快照
- **THEN** 系统返回明确的未找到错误

### Requirement: 内容树节点与边关系聚合

系统 MUST 基于种子根节点和果实父子关系聚合工作区内容树。工作区聚合结果 MUST 能表达种子节点、果实节点以及节点之间的父子边关系。

#### Scenario: 构建种子到果实的内容树

- **WHEN** 种子下存在多层果实节点
- **THEN** 系统返回从种子根节点出发的节点集合和边关系集合
- **AND** 前端能够仅基于该聚合结果构建工作区树状画布

#### Scenario: 内容树为空

- **WHEN** 种子尚未生成任何果实
- **THEN** 系统仍返回种子根节点
- **AND** 系统返回空的果实子节点关系

#### Scenario: 节点正文按需读取

- **WHEN** 工作区快照返回果实节点
- **THEN** 系统只在快照中返回画布渲染所需的果实摘要
- **AND** 系统不要求在快照中返回果实 Markdown 正文

### Requirement: 节点生长状态聚合

系统 MUST 在工作区快照中聚合节点当前生长状态，供前端展示生长动效并避免对同一节点重复发起枝化生长。

#### Scenario: 节点正在生长

- **WHEN** 某个种子或果实节点存在进行中的枝化生长
- **THEN** 系统在工作区快照中标记该节点正在生长
- **AND** 前端能够据此禁用该节点新的枝化生长入口

#### Scenario: 节点没有进行中的生长

- **WHEN** 某个种子或果实节点不存在进行中的枝化生长
- **THEN** 系统在工作区快照中标记该节点可按领域规则继续判断是否能发起生长

### Requirement: 最近失败输入提示聚合

系统 MUST 聚合节点最近失败生长输入的存在性，供前端在用户重新点击节点时恢复上次失败输入。

#### Scenario: 节点存在最近失败输入

- **WHEN** 某个节点存在最近失败的枝化生长输入
- **THEN** 系统在工作区快照中标记该节点存在可恢复失败输入

#### Scenario: 节点不存在最近失败输入

- **WHEN** 某个节点不存在最近失败的枝化生长输入
- **THEN** 系统在工作区快照中不标记该节点存在可恢复失败输入

### Requirement: 工作区资源聚合

系统 MUST 为工作区聚合悬浮输入框和枝化详情面板所需的可引用资源摘要，包括可用生成器、可引用营养内容和可引用基因洞察。

#### Scenario: 获取工作区可用资源

- **WHEN** 前端读取工作区快照
- **THEN** 系统返回当前工作区可用于枝化生长的生成器摘要
- **AND** 系统返回当前种子上下文可引用的营养内容摘要
- **AND** 系统返回当前种子上下文可引用的基因洞察摘要

#### Scenario: 排除不可引用资源

- **WHEN** 生成器、营养内容或基因洞察在其所属领域中不可用于当前工作区
- **THEN** 系统不在工作区可引用资源摘要中返回该资源

### Requirement: 工作区无私有持久化状态

系统 MUST 不为工作区聚合能力新增数据库表或保存节点拖拽位置、画布视口、布局坐标等前端交互状态。

#### Scenario: 节点被前端拖拽

- **WHEN** 用户在工作区前端拖拽节点位置
- **THEN** 系统不要求保存该节点位置
- **AND** 后续重新进入工作区时，节点布局由前端根据快照重新计算或使用前端运行时策略处理

#### Scenario: 实现工作区聚合模块

- **WHEN** 后端实现工作区聚合能力
- **THEN** 系统不新增 `docs/sql/workspace.sql`
- **AND** 系统不新增工作区数据库表

### Requirement: 工作区基因汲取中心聚合

系统 MUST 在工作区快照中聚合种子级基因汲取中心数据。该能力的接口契约 MUST 落到 `docs/api/workspace.yaml` 的 `GET /api/seeds/{seedId}/workspace` 响应中，并由 Workspace Controller 返回。

#### Scenario: 读取包含基因汲取中心的工作区快照

- **WHEN** 前端请求某个种子的工作区快照
- **THEN** 系统 MUST 返回该种子的基因汲取中心数据
- **AND** 基因汲取中心数据 MUST 包含待处理汲取提醒摘要、待确认基因建议摘要、种子级基因库摘要和可引用正式基因经验摘要

#### Scenario: 没有待处理基因汲取数据

- **WHEN** 某个种子没有待处理汲取提醒且没有待确认基因建议
- **THEN** 系统 MUST 仍返回基因汲取中心数据
- **AND** 系统 MUST 使用空集合或零值表达当前没有需要处理的汲取事项

#### Scenario: 基因汲取中心不返回正文

- **WHEN** 工作区快照返回基因汲取中心数据
- **THEN** 系统 MUST NOT 在快照中返回基因建议正文或正式基因经验 Markdown 正文
- **AND** 前端查看详情时 MUST 继续使用 `docs/api/gene.yaml` 中对应的 Gene Controller 接口

### Requirement: 工作区基因汲取数据由后端驱动

系统 MUST 由后端基于基因汲取领域事实生成工作区基因汲取中心数据，前端不得依赖自行推断来决定是否展示汲取提醒。

#### Scenario: 后端返回待处理提醒

- **WHEN** 某个种子存在待处理汲取提醒
- **THEN** 系统 MUST 在工作区基因汲取中心中返回这些待处理提醒的摘要
- **AND** 前端 MUST 能仅基于该返回结果展示汲取提示

#### Scenario: 后端返回待确认建议

- **WHEN** 某个种子存在待确认基因建议
- **THEN** 系统 MUST 在工作区基因汲取中心中返回这些待确认建议的摘要
- **AND** 前端 MUST 能仅基于该返回结果展示确认、编辑或放弃入口

#### Scenario: 后端返回基因库入口摘要

- **WHEN** 某个种子的基因库可用
- **THEN** 系统 MUST 在工作区基因汲取中心中返回基因库入口所需摘要
- **AND** 摘要 MUST 指向当前种子级基因库而非全局基因库
