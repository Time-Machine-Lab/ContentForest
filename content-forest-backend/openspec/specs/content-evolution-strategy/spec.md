## Requirements
### Requirement: 定义可迭代内容进化算法模型
系统 SHALL 定义内容进化算法模型作为枝化生长和基因汲取之间的轻量策略语义。该模型 MUST 表达算法版本、输入数据源、策略产物、基因假设和反馈调节边界，并 MUST 允许后续版本替换策略实现而不改变生成器、果实、营养库和基因库的领域归属。

#### Scenario: 使用算法版本标识策略模型
- **WHEN** 系统执行枝化生长或基因汲取 Agent 任务
- **THEN** 系统 MUST 能在任务上下文、Trace 或日志中表达当前使用的内容进化算法模型版本
- **AND** 该版本标识 MUST 不要求改变公开 API 或数据库结构

#### Scenario: 后续替换策略模型
- **WHEN** 后续版本需要优化资料选择、基因继承、变异或反馈调节规则
- **THEN** 系统 MUST 允许替换或升级策略模型版本
- **AND** 生成器 Skill、果实领域、营养库领域和基因库领域的核心边界 MUST 保持不变

### Requirement: 组织多源数据为生长策略
系统 SHALL 将来源节点、用户输入、生成器方法论、营养资料、基因经验和历史反馈组织为生长策略。生长策略 MUST 说明本轮生成目标、采用证据、继承或规避的基因、探索方向和成功判据，而不是把所有资料无差别拼接为单一上下文。

#### Scenario: 从多源上下文形成策略
- **WHEN** 枝化生长 Agent 已读取本次授权的来源节点、生成器、营养和基因上下文
- **THEN** 系统 MUST 形成可用于指导内容生成的生长策略
- **AND** 生长策略 MUST 能说明每类上下文对本次生成的作用

#### Scenario: 不使用未授权资料
- **WHEN** 某个资料或基因不在本次任务授权范围内
- **THEN** 系统 MUST 不将其纳入生长策略
- **AND** 系统 MUST 不把该资料作为候选果实的使用资源引用返回

### Requirement: 支持批量果实差异化探索
系统 SHALL 在一次枝化生长批次中支持 attempt 级差异化探索。每个果实生成尝试 MUST 拥有可区分的策略方向，系统 MUST 避免仅以同一上下文重复采样来替代多样性探索。

#### Scenario: 为多个 attempt 分配探索方向
- **WHEN** 用户请求一次枝化生长生成多个果实
- **THEN** 系统 MUST 为每个果实生成尝试分配或形成不同的探索方向
- **AND** 每个探索方向 MUST 能回溯到本次生长策略

#### Scenario: 单个 attempt 仍可使用策略
- **WHEN** 用户请求一次枝化生长只生成一个果实
- **THEN** 系统 MUST 仍然可以形成单个 attempt 的生长策略
- **AND** 系统 MUST 不要求额外的批量对比信息才能运行

### Requirement: 将基因表达为可验证假设
系统 SHALL 将基因表达为可验证的内容表达特征假设。基因假设 MUST 能表达特征内容、作用方向、适用生态位、证据来源和下一轮使用建议。

#### Scenario: 表达正向基因假设
- **WHEN** 系统从选择或优秀反馈中汲取经验
- **THEN** 系统 MUST 能将其表达为值得继承、强化、组合或变异的正向基因假设
- **AND** 该假设 MUST 包含可追溯证据来源

#### Scenario: 表达反向基因假设
- **WHEN** 系统从淘汰或低表现反馈中汲取经验
- **THEN** 系统 MUST 能将其表达为应该规避、抑制或谨慎使用的反向基因假设
- **AND** 该假设 MUST 包含适用边界，避免被错误地全局禁止

### Requirement: 保持算法模型轻量落地
系统 SHALL 在第一期以轻量方式落地内容进化策略模型。系统 MUST NOT 因本能力引入向量数据库、独立实验平台、消息队列、自动发布系统或新的必需外部服务。

#### Scenario: 不新增重型基础设施
- **WHEN** 实现内容进化策略模型第一版
- **THEN** 系统 MUST 使用现有 Agent Runtime、Tool、Skill、Trace、日志和领域服务边界完成
- **AND** 系统 MUST 不要求部署新的外部基础设施

#### Scenario: 不改变系统事实归属
- **WHEN** 策略模型生成生长策略、证据卡片或基因假设
- **THEN** 系统 MUST 保持系统事实仍由对应领域服务维护
- **AND** Agent MUST NOT 直接写数据库、写 Markdown 文件或确认基因经验

### Requirement: 基因表现参与生长策略
系统 SHALL 允许内容进化策略读取基因表现摘要，并将其作为后续枝化生长的策略参考。基因表现摘要 MUST 影响继承、强化、变异或规避建议，但 MUST NOT 替代用户输入、生成器、营养库或本轮探索策略。

#### Scenario: 高表现基因作为强化参考
- **WHEN** 某条基因经验在当前种子下拥有较多正向使用结果
- **THEN** 系统 MUST 允许生长策略将其作为值得继承或强化的参考
- **AND** 系统 MUST 保留本轮用户输入和生成器方法论的优先语义

#### Scenario: 低表现基因作为规避参考
- **WHEN** 某条基因经验在当前种子下拥有较多负向使用结果
- **THEN** 系统 MUST 允许生长策略将其作为需要规避、变异或谨慎使用的参考
- **AND** 系统 MUST NOT 将其解释为跨生态位的全局禁止规则

### Requirement: 支持基因谱系阶段性比较
系统 SHALL 支持内容进化策略基于同一种子内的谱系表现进行阶段性比较。比较结果 MUST 用于辅助策略选择，而不得自动删除或覆盖任何谱系。

#### Scenario: 多谱系表现比较
- **WHEN** 同一种子下多个基因谱系都有使用记录
- **THEN** 系统 MUST 能比较这些谱系的阶段性表现
- **AND** 系统 MUST 允许后续生长继续引用任一未归档谱系

#### Scenario: 保留探索空间
- **WHEN** 某个谱系当前表现不如其他谱系
- **THEN** 系统 MUST 保留该谱系作为可探索方向
- **AND** 系统 MUST NOT 自动从基因库中移除该谱系

### Requirement: 内容进化策略必须包含原因与证据驱动维度
系统 SHALL 要求内容进化策略显式使用用户原因、证据来源与结果表现作为策略输入。策略 MUST 能解释为什么某个特征会被保留、强化、规避或重用。

#### Scenario: 策略接收原因与证据
- **WHEN** 系统生成内容进化策略
- **THEN** 策略 MUST 参考用户原因与证据来源
- **AND** 策略 MUST 说明这些信息如何影响下一轮生成

#### Scenario: 策略可解释
- **WHEN** 用户查看内容进化策略
- **THEN** 系统 MUST 能解释保留或规避某个特征的原因

### Requirement: 内容进化策略必须支持正负向基因语义
系统 SHALL 要求内容进化策略显式区分正向基因与负向基因。正向基因 MUST 指向更高复用价值，负向基因 MUST 指向需要抑制的失效模式。

#### Scenario: 正向基因进入强化链路
- **WHEN** 策略判断某个特征属于正向基因
- **THEN** 系统 MUST 允许其进入后续强化或复用逻辑

#### Scenario: 负向基因进入规避链路
- **WHEN** 策略判断某个特征属于负向基因
- **THEN** 系统 MUST 允许其进入规避或抑制逻辑

### Requirement: 内容进化策略必须保留后续评优空间
系统 SHALL 保留对基因后续表现进行追踪、评优与再标注的策略空间。策略 MUST 支持同一特征在多轮生成中的效果比较，而不要求在本次变更中完成完整评优算法。

#### Scenario: 多轮效果可比较
- **WHEN** 同一基因特征参与多轮生成
- **THEN** 系统 MUST 保留其后续表现的比较空间
- **AND** 系统 MUST 允许后续再标注其价值变化

#### Scenario: 不要求当前完成完整评优
- **WHEN** 系统实现当前版本
- **THEN** 系统 MUST NOT 因缺少完整评优算法而阻塞当前策略落地

### Requirement: 联网研究上下文参与上下文补全层
系统 SHALL 允许内容进化策略将联网研究上下文包作为上下文补全层的一种资料来源。联网研究上下文包 MUST 被视为参考资料和证据候选，MUST NOT 替代用户输入、生成器方法论、营养库正式内容或基因经验。

#### Scenario: 生长前补充联网研究上下文
- **WHEN** 用户或系统判断本轮创作缺少平台资料、爆款案例或用户语感
- **THEN** 内容进化策略 MUST 允许引用联网研究上下文包作为补充资料
- **AND** 系统 MUST 保留该资料的来源和新鲜度信息

#### Scenario: 联网研究不替代正式上下文
- **WHEN** 本轮任务同时拥有用户输入、正式营养、基因经验和联网研究上下文
- **THEN** 系统 MUST 将联网研究上下文作为补充资料
- **AND** 系统 MUST NOT 因存在联网研究结果而忽略用户输入或正式营养引用

### Requirement: 联网研究不承担平台效果判断
系统 SHALL 明确联网研究模块只负责发现和整理外部资料，不负责判断某个内容一定会成为爆款。内容进化策略 MAY 使用联网研究结果形成创作假设，但 MUST 将真实发布验证和数据回流作为最终效果反馈来源。

#### Scenario: 使用研究结果形成假设
- **WHEN** 联网研究发现某个平台近期常见标题、封面或内容结构
- **THEN** 内容进化策略 MAY 将其转化为本轮创作假设
- **AND** 系统 MUST NOT 将该假设标记为已验证成功基因

#### Scenario: 平台效果由真实反馈验证
- **WHEN** 果实发布后产生数据反馈
- **THEN** 系统 MUST 通过数据回流和基因汲取评估表现
- **AND** 系统 MUST NOT 仅凭联网研究结果确认内容效果

### Requirement: 突变以搜索半径表达
系统 SHALL 将突变激进程度理解为内容搜索半径。突变目标 MUST 是在不偏离种子核心的前提下探索新的有效表达路线，而不是追求越新越好。

#### Scenario: 保守突变
- **WHEN** 系统使用保守突变
- **THEN** 系统 MUST 优先继承父节点方向和已验证成功基因
- **AND** 系统 SHOULD 只改变少量表达变量

#### Scenario: 均衡突变
- **WHEN** 系统使用均衡突变
- **THEN** 系统 MUST 保留种子核心和有效基因
- **AND** 系统 MAY 引入新的角度、结构或受众假设

#### Scenario: 激进突变
- **WHEN** 系统使用激进突变
- **THEN** 系统 MAY 从主简报、营养库、基因库和 Agent 联想中探索更远方向
- **AND** 系统 MUST 不违背种子事实和用户明确约束

### Requirement: 系统推荐默认搜索策略
系统 SHALL 在用户未明确选择搜索模式或突变激进程度时推荐默认值。

#### Scenario: 冷启动推荐
- **WHEN** 种子首次或缺少反馈时发起枝化生长
- **THEN** 系统 SHOULD 推荐广泛探索和均衡或激进突变

#### Scenario: 基于已选择果实继续生长
- **WHEN** 用户从已选择果实继续生长
- **THEN** 系统 SHOULD 推荐方向强化或局部变异
- **AND** 系统 SHOULD 推荐保守或均衡突变

#### Scenario: 基于负反馈重新探索
- **WHEN** 当前上下文存在明确淘汰或失败反馈
- **THEN** 系统 SHOULD 推荐负反馈规避或广泛探索
- **AND** 系统 MAY 推荐更激进的突变
