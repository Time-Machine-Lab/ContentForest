## Context

用户给出的交互方案是在工作区下方增加和基因库类似的入口，点击后打开营养库工作悬浮 Dialog。该工作台不是普通资料列表，而是种子级营养研究空间。

## Goals / Non-Goals

**Goals:**
- 提供种子工作区和种子专属营养库页面入口。
- 提供三栏式大尺寸悬浮工作台。
- 保持工作区画布上下文，不跳转到独立平台情报页。
- 保证窄屏体验可用。

**Non-Goals:**
- 不实现 Agent 研究消息逻辑。
- 不实现卡片状态业务。
- 不自行推断后端数据结构。

## Decisions

### Decision 1: 使用沉浸式 Dialog 而不是小弹窗

营养工作台包含三栏信息密度，小弹窗会压缩体验。大尺寸 Dialog 更适合持续研究和资料整理。

### Decision 2: 三栏结构固定语义

左侧是营养卡片，中间是 Agent 工作区，右侧是营养汲取建议。这样用户能够同时看见资料资产、研究过程和待处理缺口。

### Decision 3: 移动端切换为 Tab

三栏在窄屏下不可用，因此移动端以 Tab 保持同一信息架构。

## Risks / Trade-offs

- [Risk] 工作台过重影响工作区操作 → Dialog 可关闭，保留画布背景但降低干扰。
- [Risk] 三栏信息过载 → 默认中间区域最大，左右侧使用可滚动卡片列表。
