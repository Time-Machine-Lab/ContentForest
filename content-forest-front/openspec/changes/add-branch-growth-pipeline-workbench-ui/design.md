## Context

当前工作区已支持发起枝化生长、轮询任务、展示生成中状态和刷新内容树。第二期后端会把枝化生长拆成管线，并提供搜索模式、突变激进程度和路径图。前端需要把这些能力做成用户能理解的轻量控制，而不是展示算法面板。

## Goals / Non-Goals

**Goals:**

- 在枝化输入框中展示搜索模式和突变激进程度。
- 支持系统推荐默认值与用户切换。
- 展示生成中果实的临时路径图。
- 展示果实方向标签或探索摘要。
- 保持现有轮询和内容树刷新流程。

**Non-Goals:**

- 不展示复杂算法参数或数字突变率。
- 不实现后端搜索策略。
- 不做可编辑任务计划。
- 不把路径图作为果实正式内容长期展示。
- 不改变物竞天择状态规则。

## Decisions

### Decision 1: 突变使用语义化程度

前端展示保守、均衡、激进等语义选项，不展示突变率数字。

### Decision 2: 搜索模式系统推荐但可切换

默认值来自后端或前端基于契约的推荐字段。用户可在高级设置中切换，但 UI 不解释复杂算法。

### Decision 3: 路径图只挂在生成中节点

用户点击生成中节点时展示路径图。任务完成后不在果实详情第一层继续展示路径图。

### Decision 4: 管线信息保持克制

方向标签、搜索模式和路径图应帮助用户理解进度，不应把工作区变成调试面板。

## Risks / Trade-offs

- [Risk] 控件过多让输入框复杂 → Mitigation: 默认折叠到枝化生成详情，高频只显示当前推荐摘要。
- [Risk] 路径图与轮询状态不同步 → Mitigation: 路径图完全以 `GET /api/growth-tasks/{taskId}` 返回为准。
- [Risk] 后端契约未落地 → Mitigation: tasks 第一项标记依赖后端契约，未落地前不私自模拟字段进入正式实现。
