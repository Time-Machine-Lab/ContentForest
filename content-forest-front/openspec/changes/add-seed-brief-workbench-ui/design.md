## Context

当前工作区已经承载内容树、节点详情、枝化生长输入框、物竞天择、发布反馈和基因汲取入口。种子主简报是第二期新增的上下文增强能力，适合放在工作区侧栏或详情辅助区中，而不是作为新的全局页面。

前端宪法要求 API/SQL 只读，因此本 change 必须依赖后端提案先更新顶层 API 契约。

## Goals / Non-Goals

**Goals:**

- 在工作区中展示当前种子的主简报状态。
- 支持生成、查看、编辑和刷新主简报。
- 未生成状态不阻塞用户继续发起枝化生长。
- 主简报展示应轻量、紧凑、可折叠，不破坏内容树画布。

**Non-Goals:**

- 不实现主简报后端接口。
- 不修改 `docs/api` 或 `docs/sql`。
- 不做简报历史版本 UI。
- 不为果实维护完整简报 UI。
- 不把简报强制变成枝化生长前置步骤。

## Decisions

### Decision 1: 工作区内展示，不新增全局导航

种子主简报归属于具体种子，应在种子工作区中展示，不进入左侧全局导航。

### Decision 2: 手动生成入口

未生成时展示空态和“生成主简报”操作。创建种子或进入工作区时不自动生成。

### Decision 3: 面板可折叠

主简报内容可能较长，第一版以侧栏/抽屉/详情辅助面板承载，支持折叠和刷新，避免遮挡画布。

### Decision 4: 降级可用

生成、读取或刷新失败时，前端展示错误反馈，但保留内容树、节点详情和枝化生长输入框可用。

## Risks / Trade-offs

- [Risk] 简报面板占用工作区空间 → Mitigation: 默认紧凑展示摘要，可展开查看全文。
- [Risk] 用户误以为必须生成简报才能生长 → Mitigation: 空态明确表达可跳过，枝化输入框不被禁用。
- [Risk] 接口未落地导致前端阻塞 → Mitigation: tasks 中标明依赖后端契约，未更新前只做类型占位或跳过真实对接。
