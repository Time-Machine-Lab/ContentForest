## Why

营养工作台左侧需要让用户管理种子专属营养卡片，并清楚区分未沉淀、已沉淀、已归档和常驻营养。用户还需要查看卡片内容、切换状态和控制引用。

## What Changes

- 实现左侧营养卡片列表。
- 展示未沉淀、已沉淀、已归档状态。
- 支持查看营养卡片内容。
- 支持状态切换：沉淀、归档、回档。
- 支持未沉淀临时引用、已沉淀正式引用。
- 支持“常驻营养”开关，已沉淀营养可默认带入生长输入框。

## Capabilities

### New Capabilities
- `nutrient-card-lifecycle-ui`: 定义营养卡片列表、状态流转、内容查看、引用和常驻营养交互。

### Modified Capabilities

## Impact

- 影响营养工作台左侧区域和枝化生长输入框默认引用逻辑。
- 依赖后端营养卡片生命周期、常驻营养和临时引用接口。
- 不修改 API 或 SQL。
