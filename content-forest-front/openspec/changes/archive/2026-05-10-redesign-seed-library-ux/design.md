## Context

`/seeds` 当前页面已经完成种子库的基础管理闭环：未归档/已归档列表、种子详情、创建、编辑、归档、回档和打开工作区。但它的页面结构仍偏向传统“列表 + 右侧详情面板”：

- 长 Markdown 正文被压缩在右侧窄栏中，阅读宽度、行高和标题层级都不适合长文档阅读。
- `打开工作区`、`编辑`、`归档/回档` 并列展示，导致最高频动作和危险/低频动作缺少层级。
- `contentLocation`、`rootNodeId`、`updatedAt` 等系统事实混在详情底部，干扰正文阅读。
- 现有界面有较多说明型文案，不符合 `docs/spec/前端开发规范文档.md` 中“页面文案只保留用户完成当前任务所必需信息”的规则。

本设计以 `docs/design/previews/seed-library-redesign-preview.html` 为目标视觉和交互参考，并继续遵守 `docs/spec/DESIGN.md` 的 Quiet Command Workspace：深色、紧凑、低噪声、动作清晰。

## Goals / Non-Goals

**Goals:**

- 全面保留 preview 的三栏阅读工作台风格：左侧种子索引、中间宽幅阅读器、右侧目录和系统事实。
- 让种子 Markdown 文档像常规文档一样阅读，而不是塞进窄详情栏。
- 明确操作层级：打开工作区为主操作，编辑为次级操作，归档/回档为低频管理操作。
- 保持所有种子 API 调用和数据模型与 `docs/api/seed.yaml`、`docs/sql/seed.sql` 完全一致。
- 保留创建、编辑、归档、回档、打开工作区、错误反馈和加载反馈能力。

**Non-Goals:**

- 不新增、修改或删除任何后端 API。
- 不修改 `docs/sql/seed.sql` 或任何数据库结构。
- 不引入新的 Markdown 数据字段、目录字段、摘要字段或后端计算字段。
- 不实现全文搜索服务、标签系统、排序规则、批量管理或快捷键系统。
- 不改变种子领域语义：种子仍只有未归档和已归档两类状态，不提供删除。

## Decisions

### 1. 使用三栏阅读工作台替代右侧窄详情

页面主体改为：

- 左侧：种子索引区，承载搜索、未归档/已归档切换、种子卡片列表。
- 中间：种子阅读器，承载当前种子的标题、状态、主操作和 Markdown 正文。
- 右侧：辅助信息区，承载文档目录和系统事实。

这样可以让用户的主注意力落在当前种子文档上，也能保留快速切换种子的效率。替代方案是只放大原右侧详情栏，但这仍然会让种子详情像附属面板，而不是种子库的核心阅读对象。

### 2. Markdown 正文进入中央阅读器，系统事实移到右侧

`SeedDetail.markdown` 使用统一 `MarkdownViewer` 渲染，但容器应调整为文档阅读密度：更宽正文、清晰标题层级、舒适行高和稳定内边距。`contentLocation`、`rootNodeId`、`updatedAt` 等系统事实保留展示，但从正文阅读流中拆到右侧信息区。

目录可以在前端基于渲染后的 Markdown 标题生成，只作为 UI 导航辅助，不写回后端，也不要求 API 返回目录结构。

### 3. 操作层级按用户任务优先级重排

`打开工作区` 是种子库最关键的前进动作，应固定在阅读器顶部并使用主按钮样式。`编辑` 是次级动作，使用图标按钮或轻量按钮。`归档/回档` 是低频且具有状态迁移含义的管理动作，应收敛到更多菜单或低干扰管理区，不与 `打开工作区` 并排争夺焦点。

这种分层比三个按钮平铺更符合用户路径：先判断当前种子，再进入工作区；只有需要维护时才编辑或归档。

### 4. 编辑态从阅读态切换为专门编辑面板

编辑时不在阅读正文下方嵌入表单，而是将中央阅读器切换为编辑面板。保存成功后使用 `PATCH /api/seeds/{seedId}` 返回的 `SeedDetail` 恢复阅读态并同步列表可见信息；保存失败时保留用户输入并通过统一异常提示组件展示错误。

### 5. 种子索引卡片只展示扫描所需信息

左侧种子索引卡片展示标题、状态、更新时间和简短摘要。摘要可以由前端从当前已加载的 Markdown 或已有 UI 层信息中派生；如果没有详情内容，不应为了摘要新增接口字段。卡片不展示冗长系统路径，避免降低扫描效率。

### 6. 响应式降级保留阅读优先

宽屏下保留三栏；中等宽度下可隐藏右侧辅助信息区；窄屏下改为单栏顺序布局，种子索引在上、阅读器在下。无论视口如何变化，打开工作区、编辑、归档/回档和错误反馈都必须可达。

## Risks / Trade-offs

- **目录从前端 Markdown 渲染结果派生，可能受 Markdown 结构影响** → 目录只作为辅助导航；当标题不足或解析不到标题时隐藏目录或展示空状态，不影响正文阅读。
- **三栏布局在较窄屏幕空间紧张** → 使用响应式降级，中屏隐藏右侧信息区，小屏改为上下布局。
- **编辑态切换可能让用户担心阅读内容消失** → 编辑态保留当前标题和正文草稿，取消编辑时恢复阅读态，保存失败保留草稿。
- **归档/回档入口收敛后可能不够显眼** → 将其放入更多菜单或明确的管理区，并用状态文案区分 `归档种子` 和 `回档种子`，避免误触。
- **视觉调整可能影响既有静态测试** → 更新种子库相关测试，覆盖新工作台结构、主操作层级、编辑态和危险操作边界。
