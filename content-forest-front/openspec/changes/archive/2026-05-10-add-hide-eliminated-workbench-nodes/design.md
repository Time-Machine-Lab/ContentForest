## Context

工作区当前会展示工作区快照返回的全部节点，并基于这些节点计算运行时树布局与连接线。已淘汰果实虽然会弱化展示，但在节点数量变多后仍会占据树形空间，干扰用户聚焦候选果实和已选择果实。

本变更只影响前端工作区视图层。隐藏淘汰节点是运行时视图过滤，不改变果实 `eliminated` 状态，不新增后端接口，不持久化节点位置。

## Goals / Non-Goals

**Goals:**

- 在工作区 header 中新增“隐藏淘汰节点 / 显示淘汰节点”切换按钮。
- 切换隐藏状态后，使用当前可见节点重新计算树形布局。
- 隐藏淘汰节点时，同步过滤与淘汰节点相关的连接线。
- 如果当前选中节点被隐藏，自动选择种子根节点或其他可见节点，避免详情面板指向不可见节点。
- 保持节点拖拽、画布拖拽、整理树形、生长中占位节点的现有行为。

**Non-Goals:**

- 不修改果实物竞天择状态，不新增“删除”或“归档”语义。
- 不新增或修改 API、SQL、后端接口和持久化结构。
- 不保存用户的隐藏偏好，页面刷新后可回到默认展示全部节点。
- 不改变右侧详情、发布器、监控器和枝化生长接口契约。

## Decisions

1. 隐藏状态使用前端运行时布尔状态维护。

   理由：隐藏淘汰节点是工作区阅读视角，不是领域状态。将其留在前端能避免污染后端模型和 API 契约。

   替代方案：把隐藏偏好保存到后端。暂不采用，因为第一期没有用户偏好存储模型，也不应为局部视图引入持久化。

2. 树布局输入使用“可见节点集合”，源数据仍保留完整快照节点。

   理由：隐藏节点后需要重排树形，但不应丢失原始节点数据。保留完整节点集合可以在重新展示时无损恢复。

   替代方案：直接从节点数组删除淘汰节点。暂不采用，因为会导致恢复展示、轮询刷新和选中节点兜底更容易出错。

3. 连接线基于可见节点过滤。

   理由：如果父节点或子节点被隐藏，连接线继续展示会造成视觉悬空。连接线必须只连接当前可见节点。

4. 每次隐藏/展示都触发树形整理。

   理由：过滤节点会改变兄弟节点数量和子树宽度，保持旧坐标会留下空洞或拥挤。动态整理能让树始终保持清晰结构。

5. 当前选中节点被隐藏时回退到种子根节点。

   理由：隐藏淘汰节点后，详情面板不应继续展示不可见节点。种子根节点是最稳定的兜底目标。

## Risks / Trade-offs

- [Risk] 隐藏淘汰节点后，用户可能误以为淘汰节点被删除。  
  → Mitigation：按钮文案使用“隐藏/显示”，并且不改变右侧详情或物竞天择操作语义。

- [Risk] 隐藏父节点时，非淘汰子节点可能失去可见父节点导致布局不完整。  
  → Mitigation：第一期只隐藏已淘汰果实；若被隐藏节点下存在非淘汰后代，布局算法应将仍可见节点纳入可见集合，并按现有树结构尽量归入最近可见祖先或作为运行时孤立节点展示。

- [Risk] 生长任务进行中时切换隐藏状态可能导致占位节点位置变化。  
  → Mitigation：占位节点继续作为可见临时节点参与布局，切换后统一重新整理树形。
