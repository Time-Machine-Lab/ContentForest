## ADDED Requirements

### Requirement: 前端必须支持种子级研究会话恢复
前端 SHALL 在打开营养工作台时加载当前种子的营养研究会话列表，并允许用户恢复、切换和继续种子级会话。接口调用 MUST 对齐 `docs/api/nutrient.yaml` 中的营养研究会话查询契约。

#### Scenario: 打开工作台恢复最近会话
- **WHEN** 用户打开某个种子的营养工作台
- **THEN** 前端 MUST 查询该种子的营养研究会话列表
- **AND** 如果存在最近会话，前端 MUST 能加载并展示该会话的消息和可沉淀营养

#### Scenario: 新建会话后刷新页面
- **WHEN** 用户创建未绑定营养内容的新研究会话并刷新页面
- **THEN** 前端 MUST 能通过种子级会话列表重新找到该会话
- **AND** 前端 MUST 不依赖本地临时状态恢复会话

#### Scenario: 切换历史会话
- **WHEN** 用户在会话列表中选择某个历史会话
- **THEN** 前端 MUST 加载该会话详情
- **AND** 当前输入框内容 MUST 不污染被切换会话的历史消息

### Requirement: 前端必须区分种子级会话和营养内容会话
前端 SHALL 在营养工作台中区分未绑定营养内容的种子级会话与绑定某个营养内容的会话。

#### Scenario: 点击营养内容卡片
- **WHEN** 用户点击左侧营养内容卡片
- **THEN** 前端 MUST 优先加载该营养内容绑定的研究会话
- **AND** 如果该营养内容没有绑定会话，前端 MUST 展示空会话状态并允许创建新会话

#### Scenario: 点击新会话
- **WHEN** 用户点击新会话入口
- **THEN** 前端 MUST 创建或进入一个未绑定营养内容的种子级研究会话
- **AND** 页面中间区域 MUST 展示可输入的对话空态，而不是大面积无意义空白
