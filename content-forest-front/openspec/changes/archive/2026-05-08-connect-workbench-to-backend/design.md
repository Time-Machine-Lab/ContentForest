## Context

工作区页面已经完成 Mock UX，实现了内容树画布、节点详情卡片、物竞天择按钮、悬浮输入框、资源 @ 引用和生成动效。后端当前已经提供工作区快照、果实详情、种子详情和枝化生长任务接口，前端需要从本地模拟数据切换为真实接口驱动。

本次设计遵循前端模式约束：`docs/api/*.yaml` 与 `docs/sql/*.sql` 为只读顶层契约，前端只消费既有接口，不修改接口或数据结构定义。

## Goals / Non-Goals

**Goals:**

- 使用工作区快照加载种子、果实节点、边和可引用资源。
- 在前端运行时根据快照构建内容树布局，保留当前拖拽浏览体验。
- 使用种子详情和果实详情接口懒加载节点 Markdown 正文。
- 使用果实接口完成选择、淘汰、恢复候选，并同步工作区状态。
- 使用枝化生长任务接口发起生成、展示节点生长态、轮询任务结果并刷新快照。
- 将工作区资源映射到输入框 @ 引用和生长任务请求。

**Non-Goals:**

- 不实现发布器、监控器和数据回流入口对接。
- 不实现果实正文编辑保存。
- 不持久化画布节点拖拽位置。
- 不引入完整图编辑或流程编排依赖。
- 不修改 API 文档、SQL 文档或后端契约。

## Decisions

1. **新增轻量 API 模块而不是把请求写在页面内。**  
   工作区、果实和生长任务分别建立前端模块，保持与现有种子、生成器、营养库模块一致的 API 封装方式。页面只组合状态和交互，不直接散落接口路径。

2. **工作区快照作为树的主数据源。**  
   页面进入时调用 `GET /api/seeds/{seedId}/workspace`，以返回的 `nodes`、`edges`、`resources` 构建视图。节点详情 Markdown 不放在快照中，点击节点后再调用 `GET /api/seeds/{seedId}` 或 `GET /api/fruits/{fruitId}`。

3. **前端生成运行时布局，不持久化坐标。**  
   后端快照不维护画布坐标，因此前端按边关系计算层级和横向分布。用户拖拽后的坐标只在当前页面运行时生效，刷新快照时尽量复用已有坐标，但不作为业务事实。

4. **生长任务采用发起后轮询。**  
   点击生成时调用 `POST /api/growth-tasks`，将来源节点置为生长态，并轮询 `GET /api/growth-tasks/{taskId}` 直到 `completed` 或 `failed`。完成或失败后刷新工作区快照，让新增果实、失败输入和生长锁都回到后端事实。

5. **失败输入优先使用快照提示，必要时读取失败输入详情。**  
   快照中的 `failedInput` 用于提示节点存在最近失败任务；当用户需要恢复输入或重试时，前端可调用 `GET /api/growth-sources/{nodeType}/{nodeId}/failed-input` 获取完整请求信息。

6. **物竞天择操作后以接口响应和快照刷新同步状态。**  
   选择、淘汰、恢复候选分别调用果实 API。接口成功后可立即更新当前详情卡片，并刷新工作区快照确保树节点状态一致。

## Risks / Trade-offs

- **[接口失败导致工作区不可用] →** 页面提供加载失败视图和重试入口，避免静默空白。
- **[任务轮询过度频繁] →** 使用固定轻量轮询间隔，并在任务完成、失败或页面卸载时停止。
- **[刷新快照导致运行时拖拽位置丢失] →** 以节点 id 复用已有坐标；新增节点再走布局计算。
- **[后端快照资源和输入框资源模型不完全一致] →** 前端建立明确映射层，把营养库和基因统一转为 @ 引用选项，再在提交时拆分为 `nutrientRefs` 与 `geneRefs`。
- **[归档种子工作区误触发生长] →** 尊重快照 `workspaceReadOnly`，只允许查看，不允许发起新的枝化生长。
