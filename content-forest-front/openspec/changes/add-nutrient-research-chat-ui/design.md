## Context

用户希望中间区域像所有 Agent 一样可以实时交流，但 Agent 输出要分为普通沟通内容和可沉淀营养。可沉淀营养需要在对话流里特殊展示，而不是隐藏在普通文本里。

## Goals / Non-Goals

**Goals:**
- 提供 Agent 对话式研究体验。
- 特殊展示可沉淀营养块。
- 支持卡片会话切换。
- 支持模板快速填充研究问题。

**Non-Goals:**
- 不实现后端联网搜索。
- 不在前端解析非结构化 Agent 输出。
- 不让前端自行判断哪些文本可沉淀。

## Decisions

### Decision 1: 可沉淀营养块作为结构化消息渲染

前端只根据后端返回的结构化消息类型渲染特殊块，不从普通文本中猜测。

### Decision 2: 卡片会话与中间工作区强绑定

点击左侧营养卡片后，中间工作区加载该卡片会话；无会话则显示空会话状态，引导用户继续研究。

### Decision 3: 研究模板是输入辅助，不是强制流程

模板用于降低提问成本，用户仍可自由输入。

## Risks / Trade-offs

- [Risk] Agent 响应较慢 → 展示研究中状态和可取消/重试入口。
- [Risk] 用户分不清普通消息和营养块 → 可沉淀营养块使用独立视觉样式和明确操作按钮。
