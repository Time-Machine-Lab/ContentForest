## ADDED Requirements

### Requirement: 工作区果实详情接入预测图
前端 SHALL 在现有种子工作区果实详情中接入预测图视图。该接入 MUST 保持节点选择、拖拽、物竞天择、发布记录、数据回流和枝化生长输入框的既有行为不变。

#### Scenario: 果实详情中切换预测图
- **WHEN** 用户选中果实并在详情面板中打开预测图视图
- **THEN** 前端 MUST 在当前详情面板范围内展示预测图内容
- **AND** 前端 MUST 不重置画布位置、当前选中节点或枝化输入框内容

#### Scenario: 返回果实正文
- **WHEN** 用户从预测图视图切回果实正文视图
- **THEN** 前端 MUST 继续展示当前果实 Markdown 正文
- **AND** 前端 MUST 不重新选择节点或刷新整个工作区快照

### Requirement: 预测图交互不影响物竞天择
前端 SHALL 保持预测图交互与果实物竞天择操作解耦。生成、刷新或读取预测图 MUST 不改变候选、已选择、已淘汰状态展示和操作规则。

#### Scenario: 候选果实查看预测图
- **WHEN** 用户在候选果实详情中查看预测图
- **THEN** 前端 MUST 保留选择和淘汰操作的既有可用性
- **AND** 前端 MUST 不因为预测图判断自动改变按钮状态

#### Scenario: 已淘汰果实查看预测图
- **WHEN** 用户在已淘汰果实详情中查看预测图
- **THEN** 前端 MUST 保留恢复候选操作的既有可用性
- **AND** 前端 MUST 保持已淘汰果实弱化展示规则

#### Scenario: 已选择果实查看预测图
- **WHEN** 用户在已选择果实详情中查看预测图
- **THEN** 前端 MUST 继续不展示选择、淘汰或恢复候选操作
- **AND** 前端 MUST 保留发布记录和数据回流入口的既有规则

### Requirement: 预测图面板遵循工作台设计系统
前端 SHALL 让预测图面板遵循 `docs/spec/DESIGN.md` 的 Quiet Command Workspace 风格。面板 MUST 使用紧凑、可扫读、操作型的信息布局，不得使用营销式 Hero、解释页或大面积装饰图形。

#### Scenario: 渲染预测图面板
- **WHEN** 用户查看预测图
- **THEN** 前端 MUST 使用紧凑分区、状态标签、列表和工具按钮展示预测图
- **AND** 前端 MUST 不使用营销页 Hero 或与工作区不一致的视觉风格

#### Scenario: 文本不遮挡工作区操作
- **WHEN** 预测图包含较长强点、风险、反事实场景或观察指标
- **THEN** 前端 MUST 保证文本在详情面板内可读并可滚动
- **AND** 前端 MUST 不遮挡画布、顶部工具栏或底部枝化生长输入框
