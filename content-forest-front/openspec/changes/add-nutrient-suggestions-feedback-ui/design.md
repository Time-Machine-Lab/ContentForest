## Context

用户希望“枝化生长前缺资料时弹出建议”不要打断流程，而是收集到营养工作台右侧。采纳建议后，应创建未沉淀卡片并把建议内容带入中间对话区。

## Goals / Non-Goals

**Goals:**
- 展示右侧营养汲取建议。
- 支持采纳和忽略。
- 展示新鲜度提醒和使用效果摘要。
- 支持相似合并提示。

**Non-Goals:**
- 不在前端生成缺口建议。
- 不在前端计算营养效果评分。
- 不实现复杂来源可信度展示。

## Decisions

### Decision 1: 建议卡片保持轻量

建议卡片只展示缺口标题、来源和简短说明，避免把右侧做成复杂任务系统。

### Decision 2: 采纳建议联动中间输入框

采纳后创建未沉淀卡片，并将建议内容填入中间 Agent 输入框，让用户自然继续研究。

### Decision 3: 使用效果只展示摘要

前端展示引用次数、关联果实、选择/淘汰/发布反馈摘要，不把它包装成绝对评分。

## Risks / Trade-offs

- [Risk] 右侧建议过多 → 默认只展示待处理建议，支持滚动和忽略。
- [Risk] 效果摘要被误解为因果结论 → 文案强调“使用表现摘要”而不是“质量评分”。
