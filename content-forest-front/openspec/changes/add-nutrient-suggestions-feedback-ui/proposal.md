## Why

营养工作台右侧需要承载系统发现的营养缺口，并让用户快速采纳或忽略。同时用户需要看到营养的新鲜度、使用效果和相似合并提示，判断哪些资料值得继续优化。

## What Changes

- 展示营养汲取建议卡片列表。
- 支持建议采纳和忽略。
- 采纳后创建未沉淀营养卡片，并填充中间 Agent 输入框。
- 展示营养新鲜度提醒。
- 展示营养使用效果摘要。
- 展示相似营养合并提示。

## Capabilities

### New Capabilities
- `nutrient-suggestions-feedback-ui`: 定义营养建议列表、采纳忽略、新鲜度、使用回流和相似合并展示。

### Modified Capabilities

## Impact

- 影响营养工作台右侧区域、左侧卡片详情和中间输入框联动。
- 依赖后端营养缺口建议、使用回流、新鲜度和合并接口。
- 不修改 API 或 SQL。
