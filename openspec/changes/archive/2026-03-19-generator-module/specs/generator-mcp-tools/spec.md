## 新增需求

### 需求：生成器 MCP 工具注册

将生成器能力以 MCP 工具的形式对外暴露，使 AI 智能体可通过标准 MCP 协议调用生成器。

#### 场景：注册生成器工具

- **当** 系统启动或生成器列表发生变更
- **则** 将所有已激活的生成器注册为 MCP 工具
- **且** 每个工具的 `name` 取自生成器的唯一标识，`description` 取自生成器的描述信息
- **且** 工具的输入 schema 与生成器的参数 schema 保持一致

#### 场景：通过 MCP 调用生成器

- **当** MCP 客户端调用某个生成器工具，并传入符合 schema 的参数
- **则** 系统执行对应生成器的生成逻辑
- **且** 将生成结果以 MCP 工具响应格式返回
- **且** 若生成失败，则在响应中返回结构化的错误信息

---

### 需求：MCP 工具权限校验

通过 MCP 调用生成器时，需进行权限验证。

#### 场景：已授权用户调用工具

- **当** 携带有效 API 密钥的 MCP 客户端发起工具调用
- **则** 系统验证密钥对应用户是否有权使用该生成器
- **且** 验证通过后执行生成逻辑

#### 场景：未授权用户调用工具

- **当** MCP 客户端使用无效或过期的 API 密钥发起调用
- **则** 返回 MCP 错误响应，错误码为 `UNAUTHORIZED`
