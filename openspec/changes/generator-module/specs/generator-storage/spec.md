## 新增需求

### 需求：生成器持久化存储

将生成器元数据持久化保存，支持基本的增删改查操作。

#### 场景：保存新生成器

- **当** 用户提交创建生成器的请求
- **则** 系统将生成器元数据写入数据库
- **且** 返回包含自动生成 `id` 的完整记录

#### 场景：按用户查询生成器列表

- **当** 系统查询某用户的生成器列表
- **则** 返回该用户创建的所有生成器
- **且** 结果按 `updatedAt` 降序排列

#### 场景：删除生成器

- **当** 所有者发起删除操作
- **则** 系统将对应记录标记为删除（软删除）
- **且** 该生成器不再出现在任何查询结果中
- **且** 已安装该生成器的用户记录同步失效
